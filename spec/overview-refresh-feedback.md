# 概览刷新反馈

## 背景

管理工具概览页「诊断与修复」卡片中的「刷新概览」按钮点击后会执行刷新，但没有任何运行中或完成提示。用户无法判断按钮是否生效，容易误以为卡死或点击无效。

## 目标

本次要完成：

- 点击「刷新概览」后立即显示运行中反馈。
- 刷新完成后显示完成反馈。
- 保留页面切换、自动刷新时的静默行为，避免频繁弹提示。
- 增加测试锚点，防止后续改动再次丢失主动刷新反馈。

本次不包含：

- 不改变概览页数据加载范围。
- 不调整概览页布局和视觉风格。
- 不修改后端命令。
- 不改动 Codex / Claude 启动、修复、注入逻辑。

## 用户视角描述

用户在概览页点击「刷新概览」后，管理工具显示「刷新概览」正在执行；数据刷新完成后显示「概览已刷新」。如果后端调用失败，沿用现有 `run` 的失败通知。

## 功能要求

- 「刷新概览」按钮必须调用 `actions.refreshRoute("overview", { notify: true })`。
- `refreshRoute` 支持可选通知参数。
- 仅在用户主动要求通知时显示运行中和完成提示。
- 路由自动刷新仍保持静默。

## UI / 交互要求

- 运行中提示标题为「刷新概览」。
- 运行中文案应说明正在刷新概览状态。
- 完成提示应说明概览已刷新。
- 不改变现有按钮位置和样式。

## 技术约束

- 最小改动，只改前端 action 与结构测试。
- 不新增依赖。
- 不改变后端接口。

## 交付范围

- 规格文档。
- 验收标准文档。
- 前端刷新通知逻辑。
- 结构测试锚点。
- 定向验证与 manager debug 构建。
