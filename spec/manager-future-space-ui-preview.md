# 管理工具未来空间概念 UI 预览

## 背景

用户希望重新设计一版管理工具前端 UI，但明确要求先做出每个页面供评审，经用户同意后再应用到真实前端。此前直接应用概念稿造成过功能与布局回退，因此本次必须把“概念预览”和“真实应用”严格分开。

## 目标

本次要完成：

- 设计一版未来空间概念 UI 评审原型。
- 先构建“智能管理工具”的空间场景，再嵌入界面。
- 覆盖当前管理工具的 7 个页面：概览、供应商、工具与插件、会话管理、维护、设置、关于。
- 使用 3D 立体、半透明磨砂玻璃悬浮面板、局部高饱和撞色、柔和渐变光影、空间景深和高级赛博轻奢质感。
- 让用户可以逐页点击预览并评审。

本次不包含：

- 不把设计应用到真实 React/Tauri 前端。
- 不新增、删除或恢复真实应用功能。
- 不修改后端、Tauri command、注入脚本、插件逻辑、记忆逻辑或安装配置。
- 不连接真实数据。

## 用户视角描述

用户打开预览页面后，会看到一个空间化的智能管理工具控制舱。左侧或周边导航仍然能访问当前所有页面。每个页面用静态模拟数据展示真实页面的主要功能类别，使用户能评估视觉方向、布局密度、空间层次、按钮与状态表达是否合适。

## 功能要求

- 预览必须包含 7 个页面入口：概览、供应商、工具与插件、会话管理、维护、设置、关于。
- 每个页面必须展示该页面代表性的核心内容：
  - 概览：Codex 状态、Claude 状态、盘古记忆、诊断修复。
  - 供应商：供应商列表、当前配置、测试/保存类操作样例。
  - 工具与插件：Codex / Claude 插件仓库状态、MCP / Skill / 插件开关样例、插件列表。
  - 会话管理：项目会话列表、修复入口、记忆摘要。
  - 维护：入口、快捷方式、Watcher、日志类维护动作。
  - 设置：设置文件、增强矩阵、Claude 一键汉化、CLI Wrapper、运行日志摘要。
  - 关于：项目、版本、Release、社区/仓库链接样例。
- 预览导航必须是静态页面内交互，不触发真实系统命令。
- 预览页面需明确处于评审阶段，避免被误认为真实管理工具。

## UI / 交互要求

- 整体为非平面应用感：空间背景、悬浮面板、深景深、层叠光影。
- 面板为半透明磨砂玻璃，边框清晰，文字对比足够。
- 局部高饱和撞色用于状态灯、选中导航、关键按钮和数据峰值，不整页铺满高饱和色。
- 页面内容不能拥挤；面板内标题、说明、操作区要分层明确。
- 开关、按钮、徽标、列表行保持对齐。
- 响应式至少保证窄窗口下内容不水平溢出。

## 数据与接口要求

- 使用静态模拟数据。
- 不读取或写入用户配置。
- 不调用 Tauri API。
- 不发起外部网络请求。

## 技术约束

- 预览文件放在 `docs/ui-previews/future-space-manager/`。
- 可直接用浏览器打开，或通过静态文件服务预览。
- 不修改 `apps/claude-codex-pro-manager/src/App.tsx` 以应用此设计。
- 不新增 npm 依赖。
