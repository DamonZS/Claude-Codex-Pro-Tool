# 会话上下文正文显示回归修复

## 背景

“会话与记忆”页面的 Codex 与 Claude 会话共用同一个上下文弹窗。弹窗消息列表采用纵向 Flex 布局并允许滚动，但消息卡仍使用默认的 `flex-shrink: 1`。当一页包含多条或较长消息时，浏览器会把所有消息卡压缩到弹窗可视高度内；卡片同时设置了 `overflow: hidden`，最终只剩消息头部或正文第一行可见。

真实 Codex rollout `rollout-2026-07-18T02-08-51-019f7144-409c-7911-83a3-2fe3d0c890bf.jsonl` 包含 11 条可读 user/assistant 消息，所有正文均非空，而界面中多数卡片只显示角色和序号，证明正文丢失发生在展示层而不是 Codex 解析层。Claude 会话复用同一消息卡布局，因此受同一回归影响。

## 目标

- Codex 与 Claude 会话详情中的每条非空消息正文均按内容自然撑开。
- 多条或长消息超过弹窗高度时，由消息列表整体滚动，不压缩消息卡。
- 保留原有正文换行、长单词换行、角色标识、分页和关闭交互。
- 使用自动化契约测试防止消息卡恢复为可纵向收缩。
- 使用由指定真实 rollout 事件结构脱敏提炼的 JSONL 夹具，锁定多段用户输入和助手输出的正文解析。

## 非目标

- 不改变 Codex rollout 或 Claude JSON/JSONL 的解析格式。
- 不显示隐藏思考、图片内容或新的工具调用详情。
- 不修改会话列表、分页大小、删除、迁移或备份行为。
- 不修改启动器、主题中心或注入逻辑。

## 用户视角描述

用户在“会话与记忆”页面点击任意 Codex 或 Claude 会话后，弹窗按顺序展示消息。每条消息的正文完整可读；内容超过弹窗高度时，用户在弹窗正文区域滚动查看后续消息，不会看到只有角色标题的空卡片。

## 功能与 UI 要求

- `.claude-session-context-body` 继续作为唯一的纵向滚动容器。
- `.claude-session-context-message` 必须禁用纵向收缩，并保持宽度可随容器收缩，避免横向溢出。
- 消息正文继续使用 `white-space: pre-wrap`、长内容断行和自然高度。
- 不为单条消息设置固定高度、最大高度或独立滚动条。
- Codex 与 Claude 必须使用同一修复后的消息展示规则。

## 数据与接口要求

- 不改变 `CodexSessionContextPage`、`ClaudeSessionContextPage` 及其消息字段。
- 空正文仍由现有解析器过滤；前端不伪造正文或占位消息。

## 技术约束与交付范围

- 只修改会话详情相关 CSS 和对应回归测试。
- 不新增 npm 或 Rust 依赖。
- 交付独立规格、验收文档、样式修复和定向验证结果。
- 测试夹具只保留事件结构和虚构短文本，不复制真实会话正文、路径或凭据。
