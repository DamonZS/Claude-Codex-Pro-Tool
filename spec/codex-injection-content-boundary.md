# Codex 注入内容边界

## 背景

Codex 注入脚本曾复用 Claude 中文覆盖设置，在页面根节点上启动 `MutationObserver`，遍历元素和文本节点，并通过翻译词表直接写回 `nodeValue`、`textContent` 与属性。该实现无法可靠区分 Codex 原生界面文案和用户输入、项目名称、会话正文、代码或命令输出，导致英文项目内容被自动改成中文。

单词白名单只能修复个别表现，不能建立可靠的内容所有权边界。本任务要求从运行链路上禁止 Codex 注入层翻译原生页面内容。

## 目标

- Codex 注入脚本不读取 Claude 中文覆盖开关。
- Codex 注入脚本不启动通用页面文本翻译观察器。
- Codex 注入脚本不对用户输入、项目内容、会话正文、代码、路径、文件名、终端和日志执行词表替换或文本写回。
- CCP 自己创建的状态、菜单和提示继续使用既有中文文案。
- Claude 中文窗口与 Claude 本机汉化能力保持不变。

## 非目标

- 不为 Codex 增加新的界面汉化方案。
- 不修改 Claude 中文注入脚本或 Claude 本机资源补丁。
- 不调整模型选择器、插件入口、盘古记忆、状态弹窗或其他增强功能。
- 不通过新增品牌词、单词或短语例外修复问题。

## 用户视角

用户通过 CCP 启动或重启 Codex 后，可以输入任意英文、多词句子、代码、路径和项目名称。无论 Claude 中文覆盖设置是否开启，Codex 页面中的这些内容都必须保持原文，且动态新增内容同样不得被注入层翻译。

## 功能要求

1. Codex 的设置映射不得将 `claudeAppChineseOverlayEnabled` 映射为 Codex 页面翻译开关。
2. Codex 的扫描流程不得安装或刷新通用文本翻译观察器。
3. 翻译词表不得存在可到达的 Codex DOM 写回路径。
4. 修复必须覆盖初始页面和后续 MutationObserver 动态渲染内容。
5. 旧 Codex 进程必须重启后再进行验收，避免继续执行内存中的旧注入脚本。

## 技术约束

- 修改集中在 `assets/inject/renderer-inject.js` 和相关契约测试。
- 不引入新依赖，不大改管理工具 UI。
- 保留现有 CCP 自有 DOM 的中文显示。
- 默认发布构建必须输出到仓库根目录的 `target/release`。

## 交付范围

- Codex 注入运行链路修复。
- Rust 注入资源契约测试。
- 对应验收文档。
- 默认 release 目录完整构建和本地 Manager 启动。
