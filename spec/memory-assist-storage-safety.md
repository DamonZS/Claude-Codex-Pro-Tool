# 盘古记忆存储位置与写入安全

## 背景

盘古记忆数据库当前固定写入用户主目录下的 `.claude-codex-pro/memory_assist.sqlite`。在 Windows 上该目录通常位于 C 盘；同时 Codex 注入脚本可能重复提交相同采集，导致 `memory_events` 和 `memory_activity_events` 持续膨胀并占满系统盘。

## 目标

- 允许用户选择盘古记忆数据目录，并让 Manager、Launcher、MCP 使用同一配置。
- 正式安装时优先使用安装盘上的专用数据目录；开发态或安装目录不可识别、不可写时兼容回退到原应用状态目录。
- 提供无损迁移流程，迁移整个盘古记忆数据集合，而不是单独迁移 SQLite 表。
- 阻止相同采集请求重复产生诊断事件，并限制事件表体积。
- 保持现有盘古记忆页面结构，仅在高级诊断中增加紧凑的存储管理入口。

## 非目标

- 不迁移 `settings.json`、日志或其他应用状态。
- 不删除旧数据库，不删除长期记忆、候选或采集记录。
- 不重新设计盘古记忆页面或其他导航。
- 不承诺迁移期间仍在运行的旧 Launcher/MCP 进程自动切换数据库。

## 用户工作流

1. 用户在盘古记忆的高级诊断中查看当前存储目录。
2. 用户选择目标目录并确认迁移。
3. 后端校验目标可写、空间足够且不存在冲突数据库，再创建一致性数据库副本并校验完整性。
4. 迁移成功后才保存新目录设置，并提示用户重启 Codex、Launcher 和 MCP。
5. 迁移失败时继续使用原数据库，旧数据不受影响。

## 功能要求

### 路径解析

- `BackendSettings` 新增 `memoryAssistDataDir`，空字符串表示使用自动解析目录。
- 路径优先级为：测试覆盖、用户显式选择、已确认安装目录所在盘的产品数据目录、旧应用状态目录。
- 安装目录必须通过当前可执行文件及同目录产品二进制校验；不得盲目写入 `Program Files` 或仅根据 `SystemDrive` 推导。
- 目标目录必须经过可写性验证。解析失败或不可写时回退旧应用状态目录。
- `MemoryAssistStore::default()`、Manager、Launcher 与 MCP 必须使用同一 core 路径解析逻辑。

### 数据迁移

- 迁移范围包括 `memory_assist.sqlite`、SQLite 一致性数据、`pangu_memory_inject.md` 和 `memory_assist_backups`。
- 活库迁移使用 SQLite 一致性机制，不直接复制可能处于 WAL 状态的数据库。
- 目标存在不同数据库时拒绝覆盖；源目录与目标目录相同则返回无变更成功结果。
- 临时数据库必须通过 `PRAGMA integrity_check` 后才能成为目标数据库。
- 设置只能在目标数据库就绪后写入；设置保存失败时不得让默认路径切换到不完整目标。
- 成功后保留源数据库，返回源目录、目标目录、迁移文件和重启提示。

### 写入防护

- 注入脚本对采集请求按 workspace、正文和采集结果生成指纹，30 分钟内不重复提交，缓存最多 128 条；请求失败后移除指纹以允许重试。
- Core 的 `record_capture` 仅在首次插入或记录字段真实变化时写 `capture_recorded`、活动事件和注入缓存。
- `memory_events` 保留 30 天且最多 20,000 条；`memory_activity_events` 保留 90 天且最多 50,000 条。
- 裁剪按低频节奏触发并使用 `created_at` 索引，不影响长期记忆、候选、采集记录以及基于保留窗口的 7/30 天成果统计。

## UI 与交互

- 在盘古记忆默认关闭的“高级诊断”中显示当前存储目录。
- 提供“选择目录”和“迁移数据”两个明确操作；未选择新目录时禁用迁移。
- 迁移期间显示忙碌状态并防止重复提交。
- 成功、无变更和失败均使用现有通知系统；成功提示需要重启相关进程。
- 不引入新 UI 库，不修改导航、主题或其他页面布局。

## 技术约束

- 使用现有 Rust、rusqlite、Tauri、React 和 dialog 插件。
- 不记录 Token、Base URL、私密会话全文或用户绝对路径到新的诊断事件。
- 迁移操作在 blocking pool 中执行。
- 所有改动兼容旧的 `settings.json`。

## 交付范围

- Core 设置、路径解析、迁移、幂等写入、保留策略与测试。
- Manager Tauri 命令、前端类型、动作与高级诊断入口。
- Codex 注入脚本去重与契约测试。
- 默认 `target/release` 的完整本地构建产物。
