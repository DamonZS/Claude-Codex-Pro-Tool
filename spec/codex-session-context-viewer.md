# Codex 会话上下文查看器

## 背景

会话管理页中的 Claude 会话已经支持点击条目查看真实上下文、分页加载更早内容和关闭详情；Codex 会话目前只有删除操作，用户无法在管理工具内核对会话内容。

## 目标

- Codex 会话条目可点击并打开上下文详情。
- 详情交互、加载态、空态、错误态和分页行为与 Claude 会话保持一致。
- 从会话所属的受信任 Codex SQLite 数据库解析其 rollout 文件，不允许任意数据库路径读取。
- 保留现有刷新、删除、导出和项目分组行为。

## 非目标

- 不编辑、删除、迁移或恢复会话内容。
- 不改变 Codex SQLite schema 或 rollout 文件格式。
- 不展示隐藏思考链；仅展示现有 rollout 解析器可读的用户和助手文本。

## 用户视角

用户进入“会话管理”，点击 Codex 会话标题后看到覆盖层详情。详情显示会话标题、项目、数据库/rollout 来源和按顺序排列的消息；较长会话可以继续加载更早消息，并可通过关闭按钮、遮罩或 Escape 返回列表。

## 功能与交互要求

- 会话主行（删除按钮除外）可点击并支持键盘操作。
- 初次打开默认读取最近 80 条可读消息。
- “加载更早内容”向前分页，合并时按序号去重排序。
- 消息显示角色、序号、可用时间戳及正文。
- 请求失败可重试；无可读消息时显示空态；读取期间显示加载态。
- 切出会话管理路由或删除当前打开会话后关闭详情。

## 数据与接口要求

- 新增 `load_codex_session_context` Tauri 命令，输入 `sessionId`、可选 `dbPath`、`offset`、`limit`。
- 后端仅接受由 Codex home 重新发现的候选数据库路径，并通过数据库中的 thread 记录解析 rollout。
- 返回会话元数据、消息总数、当前 offset、消息页和 `hasMoreBefore`。

## 技术约束与交付范围

- 复用 `claude-codex-pro-data` 的 Codex rollout 解析逻辑。
- 文件 IO 放在 blocking task 中；分页上限应受控。
- 不新增 npm 或 Rust 依赖。
- 修改数据层、Manager Tauri 命令/注册、前端状态与 UI，并补充定向测试。
