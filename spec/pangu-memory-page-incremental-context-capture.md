# 盘古记忆独立页、能力可视化与增量采集

## 背景

盘古记忆已经具备本地采集、长期记忆、遗忘曲线、压缩整合、注入摘要缓存与 MCP 共享等后端能力，但管理工具中展示分散，概览页负担过重，会话管理页也混入记忆管理能力。用户需要一个独立的盘古记忆页面，用于看清三层链路、模块 B/C/D 能力、增量采集状态和经验教训注入手册。

## 目标

本次要完成：

- 新增独立导航 `盘古记忆`，位置在 `会话管理` 下方、`维护` 上方。
- 概览页盘古记忆只保留开关、运行状态、Codex 注入、对话监控和现有火焰特效。
- 会话管理页移除盘古记忆面板，只保留历史会话修复、Codex 会话管理和 Claude 会话诊断。
- 独立页完整展示三层链路状态、增量采集状态、模块 B/C/D 能力和来源条目审查。
- Codex 历史采集从只读用户消息扩展为读取 user / assistant / tool / system / developer 等可解析文本上下文。

本次不包含：

- 不重置或删除 `memory_assist.sqlite`。
- 不破坏 Claude 中文注入。
- 不修改概览页对话监控火焰特效。
- 不把 Claude 官方网页 LevelDB / IndexedDB 作为主采集路径。
- 不采集密钥、cookie、token 原文、二进制或不可解析缓存。

## 用户视角描述

用户进入管理工具后，可以在左侧看到 `盘古记忆` 独立入口。概览页只用于快速查看盘古记忆是否开启、Codex 是否注入、对话监控是否运行。进入独立页后，用户可以看到历史会话采集层、注入实时监听层、核心算法裁判层的状态，查看增量采集进度，阅读和编辑单个 Markdown 经验教训注入手册，查看遗忘曲线与记忆分层，并确认 MCP 跨 Agent 共享状态。

## 功能要求

### UI 与导航

- `Route` 新增 `memory`。
- 导航顺序为：概览、供应商、工具与插件、会话管理、盘古记忆、维护、设置、关于。
- 概览页不得出现经验教训、采集记录、工作区、数据库、最近备份、查看/编辑、提炼、自检等入口。
- 会话管理页不得再渲染 `MemoryAssistPanel`。

### 三层链路状态

独立页顶部必须展示：

- 历史会话采集层：Codex 源、Claude 源、采集记录数、扫描模式、最近扫描时间。
- 注入实时监听层：Codex 注入、Codex 对话监控、当前 workspace/thread、Claude 使用方式为 `MCP 共享，不走前端注入`。
- 核心算法裁判层：待处理候选、长期记忆、已归档、注入手册、自检摘要。

### 模块 B：遗忘曲线与分层

- 展示 active 记忆数、archived 记忆数、常驻豁免数、平均 strength、平均 retention。
- 来源记忆显示强度条、活跃/已归档、常驻、归档/恢复按钮。
- 文案说明归档为软标记，可恢复，注入只看 active 层。

### 模块 C：压缩整合与经验教训注入手册

- 手册是单个 Markdown 文档，不是多条卡片。
- 前端只解析 Markdown 标题生成目录，不写死业务章节内容。
- 提供重新提炼、编辑、保存、取消、复制全文、查看来源条目按钮。
- 显示更新时间、来源统计、注入缓存同步状态、生成方式。

### 模块 D：MCP 跨 Agent 共享

- 展示 `memoryAssistMcpEnabled`、`memoryAssistEnabled`、共享数据库路径、MCP 注册状态。
- 展示 Claude/Codex 使用方式。
- 展示工具能力：`memory_search`、`memory_list`、`memory_recent`、`memory_learn`。
- 提供注册 MCP 到 Claude/Codex、刷新 MCP 状态按钮。

### 增量采集

- UI 文案必须使用 `采集进度` 和 `重建采集进度`，不得使用 `cursor`。
- 后端必须维护真实 `memory_capture_progress` 采集进度表或等价状态结构，按会话源记录 `source_key`、来源类型、路径、工作区、文件修改时间、大小、扫描单位、累计捕获单位、最近扫描新增单位、最近扫描跳过标记、扫描批次和扫描时间。
- Claude 增量扫描必须覆盖 `audit.jsonl`、`local_*.json`、`.claude/sessions/*.json`、`claude-code-sessions`、`local-agent-mode-sessions` 等可解析 JSON/JSONL 会话源。
- 状态接口必须返回真实统计字段：总采集源、Codex 源、Claude 源、累计上下文、最近一次扫描新增上下文、最近一次扫描跳过未变化会话源。
- 采集层只解析可读文本、标记来源/角色/会话/工作区/顺序、脱敏、截断、幂等写入证据。
- 核心算法负责去重、留存、分类、归档、经验教训手册生成。

## 技术约束

- React/Vite/Tauri 现有架构不变。
- 不引入新 UI 库。
- 不修改 `MemoryActivityWave` 组件和火焰特效 CSS。
- 后端改动限制在盘古记忆相关链路。
- 不删除用户数据。
