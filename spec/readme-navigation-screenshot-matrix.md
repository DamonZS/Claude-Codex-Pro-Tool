# README 导航截图矩阵与能力说明更新

## 背景

当前 README 尚未完整展示 CCP 管理工具的最新信息架构，也缺少主题中心和系统提示词管理等新增能力。用户需要在仓库首页直接了解各导航页面及工具链覆盖范围。

## 目标

- 在中英文 README 中增加一致的管理工具导航截图矩阵。
- 每行固定展示四张小图，并为每张图提供明确页面名称。
- 更新功能说明，覆盖主题中心、系统提示词与指令模板管理。
- 优化仓库首屏的信息层级、关键词语义和下载转化路径。
- 保持已有安装、配置、安全边界和历史功能说明不变。

## 用户视角描述

访问仓库首页的用户可以快速浏览 CCP 的主要页面，并从功能说明了解主题应用、提示词模板管理及其他工具链能力。

## 功能要求

- 截图至少覆盖九个一级导航：概览、供应商与路由、客户端与增强、主题中心、系统提示词、会话与记忆、插件/Skills/MCP、维护与诊断、设置。
- 可以补充盘古记忆、关于与更新等页面，使矩阵完整。
- 主题系统说明应包含预览、应用、恢复默认、重启生效及安全安装策略。
- 系统提示词说明应包含内置模板、新增/编辑/删除/导入/同步、启用模式、备份与外部修改保护。

## UI / 交互要求

- 使用 GitHub README 可稳定渲染的 HTML 表格。
- 每行四个单元格，每格包含宽度一致的小图和页面名称。
- 图片应保持可辨识，不拉伸变形。

## 数据与接口要求

- 图片存放于 `docs/screenshots/` 并使用 ASCII 文件名。
- README 使用仓库相对路径引用图片。
- 中英文 README 共用同一组已脱敏截图，页面名称分别使用对应语言。
- 不新增网络接口或运行时依赖。

## GitHub 可发现性与吸引力要求

- 标题和首段自然覆盖 Codex App、Claude Desktop、Claude Code、MCP、Skills、API provider、local proxy、session、memory、theme 与 system prompt 等真实能力关键词。
- 首屏先说明产品类别、主要用户价值和支持平台，再提供下载入口与能力概览。
- 提供可点击的快速导航，帮助访客快速到达截图、功能、下载、FAQ 和开发文档。
- 使用真实截图和可验证描述建立可信度，不使用重复关键词、夸张承诺或未实现能力。
- README 只负责仓库内容相关性和访问后转化；仓库 Description、Topics、Stars、活跃度等 GitHub 仓库元数据不在本次文件修改范围内。

## 技术约束

- 不修改应用代码、Tauri 命令、配置格式或发布行为。
- 截图不得包含 API Key、Token、私密地址、个人路径、账号信息或私密会话全文。
- 不使用占位图冒充真实页面。
