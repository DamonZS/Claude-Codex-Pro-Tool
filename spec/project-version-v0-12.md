# 项目版本号统一为 V0.12

## 背景

当前 Codex 注入标识、注入弹窗和管理工具关于页仍显示内部 Cargo/Tauri 版本 `1.2.9`，而 GitHub Release 与安装资源已经使用 `V0.12`。用户要求项目对外统一使用 `V0.12`，后续每发布一次版本递增 `0.01`。

## 目标

本次要完成：
- Codex 注入标识显示 `CCP V0.12`。
- Codex 注入弹窗标题显示 `Claude Codex Pro V0.12`。
- 管理工具关于页当前版本显示 `V0.12`。
- 更新检查中的当前版本使用 `V0.12` 参与比较。
- 发布脚本继续使用 `V0.01 -> V0.02 -> ... -> V0.99 -> V1.00` 的递增规则。

本次不包含：
- 不发布新 GitHub Release。
- 不重置安装器资源。
- 不修改 Claude 中文注入。
- 不修改盘古记忆、插件仓库或供应商功能。
- 不把 Cargo/Tauri 内部 semver 强行改成非 semver。

## 用户视角描述

用户在 Codex 顶部注入标识、点开注入功能页、打开管理工具关于页时，看到的项目版本应统一为 `V0.12`，不再看到旧的 `1.2.9`。后续 release 每次自动递增 `0.01`。

## 功能要求

- 对外版本常量默认值为 `V0.12`。
- 如果 GitHub Actions 通过 `CLAUDE_CODEX_PRO_RELEASE_VERSION` 注入 release tag，则优先使用注入值。
- 没有环境变量时，不再回退显示 Cargo 包版本 `1.2.9`。
- 更新比较逻辑继续认为 `V0.12` 新于旧 semver `1.2.9`。
- 自动发布脚本的下一版本算法保持两位小数递增。

## UI / 交互要求

- 注入标识文本应为 `CCP V0.12`。
- 注入弹窗标题应为 `Claude Codex Pro V0.12`。
- 管理工具关于页版本字段应为 `V0.12`。
- 不改变现有页面布局和交互。

## 数据与接口要求

- `backend_version` 接口返回 `V0.12`。
- 注入脚本全局变量 `window.__CLAUDE_CODEX_PRO_VERSION__` 为 `V0.12`。
- 不新增接口。

## 技术约束

- 保持 Cargo/Tauri 需要的 semver 兼容。
- 不引入新依赖。
- 保持 release tag 脚本和 GitHub Actions 可验证。

## 交付范围

- 版本常量与相关测试。
- 必要的 mock/预览版本文案同步。
- 规格与验收文档。
