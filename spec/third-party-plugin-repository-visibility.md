# 工具与插件第三方插件仓库入口可见性修复

## 背景

用户反馈管理工具左侧“工具与插件”页面中，“第三方插件仓库”入口不见了。该入口属于插件中心/工具管理能力的一部分，用于访问或管理第三方插件仓库资源。入口消失会阻断用户查找、安装或管理第三方插件的工作流。

## 目标

本次要完成：

- 定位“第三方插件仓库”入口消失的根因。
- 恢复该入口在“工具与插件”页面中的可见性。
- 保持现有页面结构和功能风格，不扩大改动范围。
- 增加或更新能够防止该入口再次消失的验证。

本次不包含：

- 重做插件中心 UI。
- 改变插件安装策略或第三方仓库安全策略。
- 修改与该入口无关的供应商、会话管理、记忆或 Claude 修复功能。
- 清理仓库中已有的无关脏改动。

## 用户视角描述

用户进入管理工具后，点击左侧“工具与插件”，应能看到“第三方插件仓库”相关入口或卡片，并能继续进入第三方插件仓库管理/查看流程。

## 功能要求

- “工具与插件”页面必须展示“第三方插件仓库”入口。
- 入口文案应清晰包含“第三方插件仓库”。
- 入口应沿用当前工具/插件页面已有的卡片、按钮或分组风格。
- 若现有代码已有对应 action/tab/section，不得重新发明平行入口，应恢复既有入口绑定。
- 不得隐藏或移除已有插件中心、Ponytail、Codex 插件仓库等相关入口。

## UI / 交互要求

- 入口应位于“工具与插件”页面中，用户无需从“概览”页寻找。
- 入口不应被状态提示、滚动容器、条件渲染或响应式样式遮挡。
- 若点击入口会切换到现有详情区域，应保持原有交互方式。

## 数据与接口要求

本次不要求新增后端接口。若入口依赖现有后端状态或 Tauri command，应继续使用现有数据结构。

## 技术约束

- 优先使用现有组件、状态、类型和样式。
- 只修改与该入口可见性直接相关的文件。
- 不引入新依赖。
- 不回滚无关工作区改动。

## 交付范围

- 相关前端 UI 代码。
- 必要的测试或静态验证。
- 本规格和对应验收标准。
