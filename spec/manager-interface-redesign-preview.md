# 管理工具界面 UI 重设计评审原型

## 背景

用户要求重新设计 Claude Codex Pro 管理工具界面 UI，并点名使用 `product-design`、`awesome-design`、`awesome-design-md` 与 `frontend-design`。此前概念稿曾因为恢复旧功能、改变真实前端入口或视觉方向不满意而返工，因此本次必须先做独立评审原型，不直接应用到真实 React/Tauri 前端。

## 目标

本次要完成：

- 设计一版新的管理工具 UI 评审原型。
- 使用 Linear 风格的深色工程控制台表面层级、低噪声边框和紧凑信息密度。
- 借鉴 Cursor 风格的 AI 工作流时间线与状态标签，但不照搬品牌。
- 覆盖现有 7 个管理工具页面：概览、供应商、工具与插件、会话管理、维护、设置、关于。
- 保持功能入口可识别，布局不拥挤，长文本不遮挡按钮。

本次不包含：

- 不应用到真实 `apps/claude-codex-pro-manager/src/App.tsx`。
- 不修改 Tauri command、Rust 后端、注入脚本、插件逻辑、记忆逻辑或安装/发布配置。
- 不新增真实功能，不恢复已删除的设置页卡片。
- 不连接真实数据，不调用外部网络。

## 用户视角描述

用户打开预览后，应看到一个成熟、克制、工程感强的本地 AI 管理控制台。左侧导航保留全部页面，顶部展示当前页面目标和关键动作，主体区域用可扫描的状态卡、表格、列表、时间线和操作条表达现有功能。页面应该更像可长期使用的专业工具，而不是概念海报或营销页。

## 功能要求

- 必须包含 7 个页面入口：概览、供应商、工具与插件、会话管理、维护、设置、关于。
- 每个页面展示对应页面核心内容：
  - 概览：Codex、Claude、盘古记忆、插件仓库和诊断动作。
  - 供应商：供应商列表、当前配置、编辑表单和测试/保存动作。
  - 工具与插件：Codex 官方/第三方仓库状态、Claude 仓库状态、MCP/Skills/Plugins 开关和修复按钮。
  - 会话管理：Codex 式项目会话列表、时间线、导出、修复和记忆摘要。
  - 维护：入口、快捷方式、Watcher、后端、更新、日志。
  - 设置：设置文件、增强矩阵、Claude 一键汉化、CLI Wrapper、运行日志；不得恢复 Codex 启动参数、图片覆盖、盘古记忆、安全边界和用户脚本卡片。
  - 关于：项目说明、版本、Release、仓库和 Discord 链接。
- 预览中的按钮、开关和导航只能做静态交互，不执行真实系统动作。

## UI / 交互要求

- 深色工程控制台主题，背景低噪声，表面层级清晰。
- 卡片半径控制在 8px 左右，避免过度圆润。
- 主要按钮、状态标签、危险提示和待确认状态需有明确差异。
- 工具与插件页开关圆球必须在胶囊上下居中，并与编辑/删除/修复按钮对齐。
- 供应商编辑按钮视觉应是单只倾斜笔，不使用组合工具图标。
- 页面密度要适合运维工具，不能像营销页，也不能拥挤到难读。
- 窄窗口下内容可换行，不应出现按钮压住文本。

## 数据与接口要求

- 使用静态模拟数据。
- 不调用 `fetch`、Tauri `invoke` 或 `window.__TAURI__`。
- 不读取或写入本地配置。

## 技术约束

- 可直接用浏览器打开。
- 不新增 npm 依赖。
- 不修改真实应用代码。
