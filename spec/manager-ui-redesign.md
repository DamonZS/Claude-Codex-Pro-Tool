# 管理工具前端 UI 重设计

## 背景

当前管理工具已经承载 Codex 启动增强、供应商配置、插件仓库、盘古记忆、脚本、更新和安装维护等大量本地运维能力。随着功能增加，页面视觉层级、卡片密度、导航氛围和状态信息需要更像一个成熟的 AI coding 运维控制台，而不是普通设置页拼接。

本次重设计使用 `frontend-design`、`awesome-design-md` 和 `awesome-design` 的设计流程，主参考为 Cursor/Linear 类型的开发者工具设计语言：深色工程控制台、低噪声表面分层、清晰状态灯、紧凑但不拥挤的操作区。

## 目标

本次要完成：

- 在不重写业务逻辑的前提下，重塑管理工具整体视觉语言。
- 保留现有页面路由、功能入口、按钮行为和 Tauri 命令调用。
- 建立统一的深色控制台主题，包括背景、导航、卡片、按钮、输入、状态、表格/列表和滚动条。
- 提升工具与插件、供应商、设置等高密度页面的信息层级和可读性。
- 保持移动端和较窄窗口可用，避免文本、按钮、开关重叠。

本次不包含：

- 新增后端功能。
- 删除现有功能入口。
- 改变 Codex / Claude / 插件 / 记忆的业务逻辑。
- 引入新的 UI 依赖或图标库。
- 做营销式首页或大面积装饰插画。

## 用户视角描述

用户打开管理工具后，应感到它是一个面向 AI coding 工作台的本地控制中心。左侧导航清楚、顶部状态简洁、主体卡片层级稳定，状态灯和按钮可以快速判断哪些模块正常、待修复或正在运行。高密度列表中的开关、编辑、删除、修复、刷新等控件保持对齐，长路径和仓库信息不遮挡核心状态。

## 功能要求

- 所有既有页面仍可通过侧边导航访问。
- 现有按钮、表单、开关、状态行、插件卡、供应商卡和上下文条目仍保持原功能。
- 视觉系统必须使用 CSS 变量集中定义颜色、表面、边框、文字、半径和阴影/发光策略。
- 管理工具应默认呈现深色工程控制台主题。
- 主行动按钮、危险按钮、状态徽标和状态灯应有明确区别。
- 高密度信息卡不应依赖大面积阴影或花哨渐变表达层级。
- 页面顶部不再显示后端链接胶囊。

## UI / 交互要求

- 主布局：侧边栏 + 顶部条 + 内容区保持现有结构，但整体更像本地运维控制台。
- 侧边栏：当前路由有明显选中态；导航项间距稳定；图标和文字不重叠。
- 内容区：卡片半径不超过 8px，边框清晰，内部标题、描述、操作区层级明确。
- 按钮：使用 lucide 图标时保持图标与文本垂直居中；小图标按钮尺寸稳定。
- 状态：使用小型状态灯/徽标表达 `ok`、`running`、`needs_review`、`error`、`not_checked` 等状态。
- 表单：输入、选择、文本域、开关必须在深色主题下有足够对比度。
- 列表：长路径、仓库 URL、命令预览允许换行或截断，但不得遮挡按钮。
- 工具与插件页：MCP / Skills / 插件条目右侧开关必须在固定操作槽内水平与垂直居中，滑块不得偏出轨道或与编辑、删除按钮错位。
- 供应商页：供应商列表中的编辑按钮必须使用单只倾斜笔图标，不使用交叉工具或组合工具图标。
- 响应式：窄窗口下侧栏和内容区不能横向溢出到不可操作状态。

## 数据与接口要求

- 不修改 Tauri command 名称、参数和返回结构。
- 不修改用户配置文件写入逻辑。
- 不新增远程资源或外部字体请求。

## 技术约束

- 优先改 `apps/claude-codex-pro-manager/src/styles.css`。
- 只在必要时小范围调整 `App.tsx` 的类名或文案。
- 不新增 npm 依赖。
- 不修改 `assets/inject/claude-chinese-inject.js`。
- 不修改与本次 UI 无关的 Rust 后端逻辑。
