# 工具与插件上下文管理切换与开关布局修复

## 背景

用户反馈在“工具与插件”页中，`Codex 工具与插件` 和 `Claude 工具与插件` 面板的 `MCP`、`Skills`、`插件` 标签切换会卡顿，并且下方条目里的开关样式出现重叠、显示不正确。

## 目标

本次要完成：

- 修复上下文条目行里开关被其它按钮样式覆盖导致的重叠问题。
- 降低 `MCP`、`Skills`、`插件` 标签切换时的重复计算和不必要重排。
- 保持现有 MCP、Skill、Plugin 数据结构、保存逻辑和按钮行为不变。

本次不包含：

- 重做工具与插件页面信息架构。
- 修改供应商切换、配置写入或插件安装策略。
- 清理仓库中已有无关改动。

## 用户视角描述

用户进入“工具与插件”页，在 `Codex 工具与插件` 或 `Claude 工具与插件` 面板中点击 `MCP`、`Skills`、`插件` 标签时，页面应快速切换列表。列表条目右侧的启用开关应显示为正常滑动开关，不能与编辑、删除按钮重叠。

## 功能要求

- `MCP`、`Skills`、`插件` 三个标签仍显示各自条目数量。
- 标签切换只改变当前列表和说明文案，不触发额外保存或刷新命令。
- 条目启用开关仍可用于可编辑条目的启用/禁用。
- 编辑和删除按钮仍保留原有行为。

## UI / 交互要求

- 条目行右侧应稳定排列为：启用开关、编辑按钮、删除按钮。
- 启用开关应保持固定宽高，不被图标按钮样式覆盖。
- 长标题或长摘要不能挤压开关、编辑按钮或删除按钮。
- 禁用态仍应可辨识，但不能造成布局塌陷或重叠。

## 数据与接口要求

本次不新增后端接口，不改变现有 Tauri command 入参和返回结构。

## 技术约束

- 优先使用现有 React state、CSS class 和组件。
- 不引入新依赖。
- 不改动插件安装、MCP 写入、Claude Desktop 开发模式等后端逻辑。

## 交付范围

- `apps/claude-codex-pro-manager/src/App.tsx` 中上下文管理面板的必要渲染优化。
- `apps/claude-codex-pro-manager/src/styles.css` 中上下文条目行和按钮样式修复。
- 对应验收标准和验证记录。
