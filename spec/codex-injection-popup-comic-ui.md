# Codex 注入弹窗漫画质感 UI

## 背景

用户希望 Codex 顶部注入标识点开后的功能页面不再是普通白色表单风格，而是改成更有辨识度的漫画质感 UI。该弹窗承载 Claude Codex Pro 的功能开关、推荐内容、支持入口、关于与服务模式等现有能力。

## 目标

本次要完成：
- 将 Codex 注入弹窗主视觉改为漫画质感：粗描边、纸张网点、气泡标题、卡片分层、漫画阴影和高饱和强调色。
- 保留现有弹窗结构、tab、开关、按钮、服务模式、推荐内容、支持、关于等功能。
- 保持弹窗可滚动、可关闭、可点击，不影响 helper bridge 与盘古记忆注入。

本次不包含：
- 不新增或删除功能入口。
- 不修改 Claude 中文注入。
- 不修改盘古记忆数据库、候选生成或长期记忆逻辑。
- 不重构管理工具主界面。

## 用户视角描述

用户点击 Codex 顶部 `CCP` 注入标识后，看到的是一个漫画风格的功能控制面板：视觉更活泼，边缘有漫画描边和阴影，内容区仍然紧凑清晰，所有原按钮与开关仍可正常使用。

## 功能要求

- 弹窗仍通过原 `openClaudeCodexProModal()` 打开。
- `主页`、`推荐内容`、`支持` 三个 tab 保持存在。
- 所有 `data-claude-codex-pro-setting` 开关保持可用。
- 服务模式按钮、宽度输入、后端修复按钮、Discord/问题按钮保持可用。
- 弹窗内容仍可滚动，不得遮挡关闭按钮。

## UI / 交互要求

- 弹窗使用漫画质感外观：粗边框、偏移投影、网点/纸纹背景、角标或装饰色块。
- 不使用密集花哨到影响阅读的背景。
- 标题区域强调品牌名和连接状态。
- 行项目保留清晰分隔，但可改成漫画卡片式分隔。
- 开关和按钮适配新主题，但尺寸与点击区域不能变小。

## 数据与接口要求

- 不新增接口。
- 不修改 `/settings/get`、`/settings/set`、`/backend/status` 等 bridge 路由。

## 技术约束

- 只改 `assets/inject/renderer-inject.js` 及必要测试/文档。
- 不引入新依赖。
- 必须通过现有注入脚本相关测试、前端检查和构建。

## 交付范围

- 注入弹窗 CSS/样式调整。
- 回归测试。
- 规格与验收文档。
