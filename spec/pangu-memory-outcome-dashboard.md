# 盘古记忆成果看板

## 背景

现有盘古记忆页完整暴露采集、注入、遗忘曲线、手册、MCP 和来源审查，适合诊断但不利于用户快速确认“今天记住了什么、Agent 是否真实召回过、当前项目有哪些关键规则”。本任务在不删除既有能力的前提下，将默认视图改为成果导向，并补齐可审计的活动事件。

## 目标与非目标

### 目标

- 面向个人与多 Agent 开发者，优先支持项目接续、历史修复召回、偏好/规范复用和跨 Agent 接力。
- 默认展示项目接续、今日成果、最近召回证据、待确认记忆和项目记忆。
- 提供 7 天/30 天真实趋势及项目、类别分布。
- 真实记录搜索、会话注入和 MCP 召回来源；展示时间、来源、查询摘要和命中记忆。
- 将采集链路、手册、遗忘参数、MCP 注册、导入导出放入默认折叠的高级诊断区。

### 非目标

- 不实现复杂知识图谱或新的向量模型。
- 不重做导航、概览页或其他管理页面。
- 不宣称系统能够证明某个错误已被避免；只能陈述某条历史修复或规则被真实召回/注入。
- 不删除、迁移或自动清理用户记忆。

## 用户视角

用户进入盘古记忆页后，首先看到当前项目、最近更新、关键规则和最近进度；可复制项目接续摘要给 Agent。随后看到今天新增采集、长期记忆、待确认和真实召回次数。若 Agent 通过 Codex 会话注入、MCP 搜索或管理器搜索命中记忆，页面展示可追溯证据；没有事件时明确显示尚无可验证记录。

## 功能要求

### 活动事件

- 新增独立活动事件存储，至少支持 `search`、`inject`、`learn`、`candidate`、`capture`、`archive`、`restore`。
- 事件包含：事件类型、查询来源 workspace、Agent 来源、可空 memory id、脱敏且截断的查询摘要、可空会话来源、创建时间、非敏感聚合元数据及命中时的脱敏限长记忆快照。
- 搜索只有真实命中时才形成召回证据；普通列表读取不得计为召回。
- Codex 会话摘要/注入与 MCP `memory_search` 必须标记真实来源；管理器搜索标记 manager。
- 事件写入采用尽力而为策略，失败不得使主查询、注入或 MCP 调用失败。
- 查询内容、来源会话和元数据写入前复用现有脱敏能力；查询摘要限制长度。
- 召回事件按请求来源 workspace 严格隔离；命中 global 记忆时仍归属于发起查询的项目，其他项目及 global 看板不得看到该项目的查询摘要。
- 最近召回证据读取命中时快照，不与可编辑、可移动的当前记忆联表；旧版无快照事件显示不可用，不展示后来修改的正文。

### 成果聚合

- 提供独立看板接口，支持 7 天和 30 天范围。
- 返回今日采集数、今日新增长期记忆、当前待确认数、今日召回数。
- 返回逐日采集/新增/召回趋势、项目分布、类别分布和最近召回事件。
- 返回当前 workspace 的项目接续条目，优先项目规则、历史修复、进度/决策及最近更新内容。
- 聚合必须来自 SQLite 真实记录，不从前端状态推测。

## UI / 交互要求

- 默认首屏依次为：项目接续卡、今日记忆成果、7/30 天趋势、最近 Agent 召回证据、待确认、项目记忆。
- 项目接续卡提供“复制给 Agent”。
- 召回证据显示来源、方式、时间、查询摘要、命中记忆；空态使用“尚无可验证召回记录”。
- 不使用“已经避免错误”等不可证明措辞，可使用“已召回上次修复记录”。
- 项目记忆保留搜索、编辑、归档/恢复能力。
- 既有三层链路、增量采集、注入手册、遗忘曲线、MCP 和导入导出保留在一个默认闭合的原生折叠区。
- 不引入新 UI 库，不修改火焰特效组件。

## 数据与接口要求

- Rust 核心公开 `MemoryOutcomeDashboard`、`MemoryActivityEvent`、趋势与分布结构。
- Tauri 增加独立看板命令；失败返回结构完整的空看板。
- MCP 调用通过核心公开的来源化查询接口记录 `mcp`，避免在 MCP 层复制数据库逻辑。
- 活动事件不纳入现有记忆导出正文，避免意外导出查询历史；后续清理策略另行设计。

## 技术约束

- 保持 Rust + Tauri + React/Vite 架构。
- 后端改动限于盘古记忆核心、MCP 适配和 Tauri 命令。
- 做最小 UI 重排，不修改其他路由与页面。
- 不记录 API key、通用 token/password/secret 赋值、Bearer/Basic 凭据或完整授权材料。

## 交付范围

- 本规格与匹配验收文档。
- SQLite 活动事件、聚合模型、来源化召回接口及测试。
- Tauri 看板命令与前端类型/状态接线。
- 简化后的盘古记忆成果页及默认折叠高级诊断。
- 定向 Rust 测试、前端检查/构建和 manager 契约测试证据。

## 新项目启动指南

- 在“项目接续”之外提供“开启新项目”场景，从所有历史 workspace 与 global 的活跃记忆中提取可泛化经验。
- global 经验可独立入选；非 global 经验只有在至少两个不同私有 workspace 的相似内容共同印证后才可跨项目入选，单个私有项目独有经验必须排除。
- Core 提供只读 `MemoryAssistStore::new_project_guide()`；结果包含稳定的 `generated_at`、参与精选的唯一 `source_item_count`、不暴露名称的 `source_workspace_count`、避坑经验、优秀做法和完整提示词。
- 指南只读、完全本地且确定性生成，不调用第三方 API，不计为 Agent 搜索或注入召回。
- 优先使用 `lesson-learned`、`lesson-manual`、`safety-rule`、`project-rule`，复用现有脱敏、去噪、行动句评分和去重逻辑。
- 排除归档项、绝对/相对脚本路径、Token/Key/Password/Secret、敏感 Base URL、全库任一 workspace 或会话标识、项目专属命令与无法泛化的上下文。
- 最多返回 12 条“优先避坑”和 12 条“优秀处理方式”；每条只暴露文本、来源数量与类别，不暴露 workspace 名称或会话 ID。
- 完整启动提示词固定要求 Agent：先读项目说明/规格/验收/源码；实施前总结目标、改动、禁区、验收与风险；最小修改；先建立可复现验证；完成后提供真实证据；按新项目规则调整历史经验。
- 完整启动提示词同时插入本次筛选出的避坑经验和优秀做法；空态不插入或虚构历史经验。
- 无合格经验时返回结构完整的真实空态，不生成虚构经验。

## 易用化补充

- 首屏“开始工作”使用双场景卡：继续当前项目、开启新项目。
- 新项目指南采用“预览后复制”，预览包含优先避坑、优秀处理方式和完整启动提示词，不提供常驻编辑器。
- 默认页面压缩为开始工作、今日成果、趋势与分布、最近召回与待确认四个主要区域。
- 项目记忆管理保持搜索、编辑、归档/恢复，但默认紧凑展示并允许展开。
- 三层链路、增量采集、注入手册、遗忘曲线、MCP、导入导出、自检和来源详情统一置于默认关闭的高级诊断。
- 窄屏下双卡与图表自动切换为单列；不引入新 UI 库，不修改其他页面或火焰特效。
