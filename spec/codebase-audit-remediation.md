# 全量代码审查缺陷修复

## 背景

全工作区格式检查、前端类型检查、前端构建和 Rust 测试均可通过，但安全边界、发布产物、异步状态与注入脚本仍存在测试未覆盖的缺陷。本任务修复有明确代码证据或可复现路径的问题，并补充回归契约。盘古记忆当前还会被频繁 DOM 变更重复触发采集；Core 虽对 `memory_captures` 做了 upsert，但仍会为相同采集反复追加事件，且事件表没有时间或数量上限。记忆数据库固定落在用户目录，也缺少 Manager、Launcher、MCP 共用的可选存储目录和无损迁移流程。

## 目标

- 收紧应用更新下载与启动的信任边界，Renderer 不能指定任意安装包。
- 避免供应商配置预览泄露 Bearer Token，并明确生成文件不可直接编辑。
- 保证 Unix 敏感配置和备份以仅当前用户可读写的权限创建。
- 缩小 Claude 汉化提权写入权限，不向本机 Users 组递归授予 Modify。
- 让 Windows、macOS 正式包包含启动器、Manager 和盘古记忆 MCP 三个运行时二进制。
- 修复自动发布复用旧 draft tag、取消不清理和 `latest.json` 使用 draft URL 的问题。
- 修复后端修复命令误报成功、同步启动阻塞 UI、设置草稿和记忆搜索的异步竞态。
- 修复 Claude 汉化替换顺序、重复注入资源累积、模型弹窗误识别和 iframe 状态层遮挡。
- 阻止盘古记忆因重复 DOM 扫描和重复 Core 请求无上限写入 SQLite，并为短期事件表建立明确保留上限。
- 将完整盘古记忆 SQLite 默认放在安装盘的可写持久数据目录，允许用户选择其他目录，并让 Manager、Launcher、MCP 始终解析到同一数据库。
- 提供带完整性校验、冲突保护和原子切换的数据库迁移，不丢失长期记忆、待确认候选或采集记录。

## 非目标

- 不重做页面布局、导航、视觉主题或供应商配置格式。
- 不删除用户数据，不改变已有长期记忆、待确认候选或采集记录的业务内容；仅按本规格迁移完整数据库和清理有界的事件审计表。
- 不为纯 Clippy 风格告警做全仓重构。
- 不承诺依赖真实 Claude Desktop、GitHub 发布权限或平台签名服务的操作百分之百成功。

## 用户视角

- 用户在“关于”页安装更新时，只会下载本仓库当前版本索引中与本机平台匹配的安装包。
- 隐藏凭据时，供应商生成配置不会展示 Token；生成配置以只读预览呈现。
- 官方安装包安装后，盘古记忆 MCP 可从 Manager 同目录找到。
- 启动、保存设置和搜索记忆期间，较晚发起的用户操作不会被旧响应覆盖。
- Claude 汉化不会生成“安装ed”一类混合文本，重复聚焦窗口不会累积观察器和定时器。
- 同一条消息引起的连续 DOM mutation 不会重复生成盘古采集事件；应用重启或并发重试也不会绕过 Core 幂等保护。
- 新安装默认把盘古记忆放在安装盘；用户可在 Manager 查看有效路径、选择目录并迁移，迁移成功后会收到重启 Manager、Launcher、MCP 相关进程的明确提示。

## 功能要求

### 更新信任

- `perform_update` 只接受前端提交的预期版本标识。
- Manager 后端必须从固定 `DEFAULT_LATEST_JSON_URL` 重新获取发布信息，并确认版本一致。
- Core 必须拒绝非 HTTPS、非 `github.com`、非固定仓库 release download 路径、非当前平台预期安装器名称的下载地址。

### 配置与文件权限

- `experimental_bearer_token` 及常见 Bearer/API Key 字段在隐藏凭据时必须脱敏。
- Header/Body 请求覆盖在隐藏凭据时必须脱敏并只读，避免编辑过程把脱敏占位符写回配置。
- Codex `config.toml` 和 `auth.json` 生成内容为只读预览。
- Unix 原子写入的敏感文件和实时备份文件权限为 `0600`，新建敏感目录权限为 `0700`。
- Windows Claude 汉化流程不得向 `S-1-5-32-545`（Users）递归授予 Modify。

### 发布

- Windows stage、ZIP、NSIS 安装/卸载均包含 `claude-codex-pro-mcp.exe`。
- macOS Manager App 的 `Contents/MacOS` 包含并签名 `claude-codex-pro-mcp`。
- 复用 draft release 前必须确认 tag 指向当前 `${{ github.sha }}`；不一致时删除 draft/tag 后重建。
- 失败或取消均清理未发布 draft；`latest.json.url` 固定由仓库与 tag 构造，不含 `/untagged-`。

### 命令与前端状态

- `repair_backend` 在 wrapper 修复失败时返回失败状态并保留设置 payload。
- `launch_claude_codex_pro` 的路径和进程探测在 blocking pool 执行。
- 设置请求仅在请求期间草稿未变化时回写草稿。
- 设置刷新与保存并发时，仅允许最后发起且期间未被用户编辑的请求回写草稿。
- 记忆搜索仅允许最新请求更新结果。
- Manager 会话删除测试必须把备份写入临时目录，不依赖或污染用户真实应用状态目录。

### 注入

- 文本替换按更具体、更长的源文本优先。
- Claude 汉化脚本重复执行时复用单个运行实例，不重复注册 observer、timer 与 listener。
- 模型增强仅注入结构可确认的模型选择弹窗，普通包含 `Model` 文本的 dialog 不注入。
- iframe fallback 在加载成功后隐藏，且展示期间不截获 iframe 交互区域点击。

### 盘古记忆写入去重与 Core 幂等

- `MutationObserver`、`scanDeferred`、聚焦、路由刷新或定时扫描在短时间内反复看到同一会话 DOM 时，必须合并扫描并以“工作区 + 会话标识 + 最新用户消息的规范化内容哈希”为采集指纹；指纹未变化时不得再次调用 `/memory/capture` 或触发等价的自动候选采集。
- 注入层必须同时记录“正在处理”和“最近成功处理”的指纹：相同指纹的并发扫描复用或跳过在途请求；只有消息、会话或工作区变化后才允许产生新的采集。失败请求不得被永久标记为成功，可在有界退避后重试。
- 注入去重只用于降载，不能成为唯一一致性保证。`MemoryAssistStore::record_capture` 必须以数据库事务和持久唯一身份实现幂等，不能依赖单进程内存状态。
- 对规范化后完全相同的采集请求，串行重放、并发重放或不同运行时重放都只能保留一条 `memory_captures` 记录、一条对应的 `memory_events.capture_recorded` 和一条对应的 `memory_activity_events.capture`。重复请求不得仅为刷新时间戳而制造写放大。
- 同一采集身份的业务字段确有变化时允许更新采集记录并追加一次可审计变更；采集记录更新与两个事件表写入必须处于同一事务，不能留下“采集成功但事件部分写入”的状态。

### 事件保留与空间边界

- `memory_events` 只保留最近 30 天，且全库最多 20000 条。清理时先删除 `created_at < 当前时间 - 30 天` 的记录，再按 `created_at DESC, id DESC` 保留最新 20000 条。
- `memory_activity_events` 只保留最近 90 天，且全库最多 50000 条。清理时先删除 `created_at < 当前时间 - 90 天` 的记录，再按 `created_at DESC, id DESC` 保留最新 50000 条。
- 保留清理必须在数据库打开/迁移后以及事件追加路径中自动、幂等地触发；实现可做节流，但持续写入时不能让表无限超过数量上限。清理和当前事件写入必须保持事务一致性，并使用索引或等价的有界 SQL，避免全量记录逐条加载到内存。
- 上述保留策略只作用于 `memory_events` 和 `memory_activity_events`。不得删除、截断、重建或按年龄清理 `memory_items`、`memory_candidates`、`memory_captures`，也不得关闭长期记忆候选生成与采集能力。

### 统一存储目录与路径解析

- 新安装的有效数据库路径必须位于安装盘上的用户可写持久数据目录，固定文件名为 `memory_assist.sqlite`；不得继续默认写入 `~/.claude-codex-pro/memory_assist.sqlite`。不得把数据库写进 macOS 签名 App bundle 或其他只读安装内容目录。
- “整个记忆数据库”包括 SQLite 主文件及其 `-wal`、`-shm`、journal、迁移临时文件和数据库备份；这些文件必须位于所选数据目录内，不能把不同组成部分拆到用户目录和安装盘两处。非数据库型设置与日志不在本条迁移范围内。
- Manager 提供当前有效路径、安装盘默认目录和用户自选目录。自选值是绝对目录，数据库文件名仍由系统管理；保存前必须完成规范化、可创建/可写检查，并拒绝文件路径、当前数据库所在目录、危险系统目录和无法稳定解析的目标。
- Core 提供唯一的共享路径解析器，以共享设置中的持久目录选择和安装位置为输入。Manager、Launcher、盘古记忆 MCP 及其启动的 helper 都必须经该解析器打开数据库，禁止各自维护默认值、复制路径或静默回退到旧用户目录。
- 安装盘默认目录不可用时，应返回可操作错误并引导用户选择可写目录；不得静默创建第二份数据库。仅测试可使用显式临时路径覆盖，生产运行时不得受测试覆盖入口影响。
- 已有用户若仅存在旧路径数据库，所有运行时在迁移完成前继续共同使用旧库并将状态标记为“待迁移”，不得在新默认目录创建空库形成分叉。Manager 必须提供迁移到安装盘默认目录或用户所选目录的入口。

### SQLite 无损迁移

- 选择新目录后，如果已有数据库，Manager 必须执行数据库迁移而不是只改路径指针。迁移前显示源路径、目标路径、预计操作和重启要求，并要求所有会写盘古记忆的 Launcher/MCP/helper 退出或进入共享迁移锁。
- 迁移必须取得跨进程排他迁移锁并阻止新写入；无法确认写入已静止时直接失败，保持旧库为唯一活跃库。
- 预检必须确认源库可读且 `PRAGMA integrity_check` 返回 `ok`，目标目录可写，源和目标不是同一文件，并且最终目标 `memory_assist.sqlite` 不存在。目标文件、目录项或大小写等价路径存在时均视为冲突，绝不覆盖、合并或截断。
- 迁移必须使用 SQLite `VACUUM INTO` 将源库写入目标目录内本次操作独占的临时文件，确保 WAL 中已提交数据进入一致快照；禁止用普通文件复制代替 SQLite 快照。
- `VACUUM INTO` 完成后必须重新打开临时库，要求 `PRAGMA integrity_check` 为 `ok`，校验 schema/user_version，并至少核对 `memory_items`、`memory_candidates`、`memory_captures` 的行数。任一检查失败均不得切换路径。
- 通过检查后，先持久化临时文件，再在目标目录内以“不替换既有文件”的原子 rename 提交为 `memory_assist.sqlite`，最后原子写入共享路径设置。只有目标库提交和设置提交都成功，迁移才可报告成功。
- 任一步骤失败时，共享路径设置保持旧值，旧数据库及其 WAL/SHM 不得删除、改名或截断；只能清理本次创建且能够证明归本次操作所有的临时/未提交目标。若设置提交失败，必须回滚本次目标提交或保持其为明确的非活跃恢复产物，不能让两个运行时分别选择不同库。
- 迁移成功后旧源库默认保留为非活跃回滚来源，不自动删除。界面必须显示新路径并提示重启 Manager、Launcher、盘古记忆 MCP 及相关 helper；重启完成前不得宣称所有运行时已切换，也不得恢复跨进程记忆写入。
- 迁移过程不得删除或重新生成长期记忆、待确认候选和采集记录；事件表可以在目标库完整性检查通过后按本规格保留策略清理。

### 存储目录交互状态

- 路径选择与迁移必须有明确的空闲、预检、迁移中、成功待重启、目标冲突、占用和失败状态；迁移中禁止重复提交。
- 取消目录选择或预检失败不得更改共享路径。失败消息必须说明阶段和可恢复动作，但不得输出记忆正文或其他敏感内容。
- 新安装且没有源数据库时，可在校验后的目标目录原子创建新库；已存在源数据库时禁止以“新建空库”绕过迁移。

## 技术约束

- 沿用 Rust、Tauri、React/Vite 和现有脚本，不新增 UI 或网络依赖。
- 更新校验使用结构化 URL 解析，不依赖字符串前缀近似判断。
- 现有 Tauri `CommandResult` 契约和设置 payload 字段保持兼容。
- 所有修改保持最小范围，并为安全或发布状态机补充自动化断言。
- 路径选择必须存入 Manager、Launcher、MCP 均可读取的现有共享设置体系；数据库不能反向保存其自身位置。
- SQLite 幂等、保留清理、`VACUUM INTO`、完整性检查和活动路径切换集中在 Core，UI 与各二进制不得重复实现数据库迁移算法。
- 数据迁移不得用覆盖式 rename、先删目标再 rename 或“复制成功即删除源”的流程；原子性必须覆盖目标提交和活动路径切换的失败恢复。

- Core 更新、设置、relay 配置和 Claude 汉化补丁实现及测试。
- Manager Tauri 命令、前端状态和供应商预览实现及测试。
- Codex/Claude 注入脚本及契约测试。
- 盘古注入扫描指纹去重、Core 采集幂等和事件表保留策略及其并发/边界测试。
- 共享存储路径解析、Manager 目录选择与迁移状态、Launcher/MCP 路径一致性，以及 SQLite 成功/冲突/故障迁移测试。
- Windows/macOS 安装脚本、自动发布工作流和发布契约测试。
- 默认 `target/release` 的完整本地构建产物。
