# 工作区脏数据与代码清理

## 背景

当前工作区存在大量既有脏改动，其中包含功能代码、规格文档、测试改动，以及明显误入仓库的本地运行数据。用户要求处理脏数据和代码，但同时强调工作区本身已有很多既有脏改动，不能盲目回滚。

本任务的目标是把可确认的垃圾数据从 Git 工作区中清理出去，并用真实检查确认现有代码不会因为脏改动出现基础格式、类型或构建问题。

## 目标

本次包含：

- 清理可确认不应纳入仓库的本地运行数据。
- 为同类运行数据补充忽略规则，避免再次进入 Git 状态。
- 检查当前代码是否存在冲突标记、空白错误、格式错误、前端类型错误或 Rust 构建错误。
- 只修复会破坏构建或检查的代码问题。

本次不包含：

- 回滚无法确认归属的既有功能改动。
- 删除用户本地 Codex、Claude、盘古记忆、供应商或登录配置。
- 重置 Git 历史、清空工作区或批量恢复到某个提交。
- 调整产品功能范围或重新设计页面。

## 用户视角

用户继续在当前工作区开发时，`git status` 不应再出现明显的本地运行垃圾文件。当前代码应通过最基础的格式和构建检查，方便后续继续修复功能问题或整理提交。

## 功能要求

- `Files/` 目录下的本地运行痕迹不得进入 Git 跟踪范围。
- 已经误入工作区的 `Files/WindowsApps/Claude_1.15962.0.0_x64__pzs8sxrjxfjjc` 必须移出 Git 状态。
- 不得删除或回滚现有已跟踪源码改动。
- 不得把 API key、Bearer token 或授权材料写入日志、文档或测试。
- 若检查发现代码问题，只允许做最小必要修复。

## UI / 交互要求

本任务不涉及 UI 改动。

## 数据与接口要求

- 不读取或修改用户本地生产配置。
- 不修改 `~/.codex/`、`%APPDATA%/Claude`、`%LOCALAPPDATA%/Claude-3p` 或 `~/.claude-codex-pro/` 下的数据。

## 技术约束

- 遵守 `AGENTS.md`，先有规格和验收，再进行清理。
- 使用 Git 状态、格式检查、前端检查和 Rust 构建作为验证证据。
- 删除文件必须限定在仓库工作区内，不能递归删除不确定路径。

## 交付范围

- `.gitignore`
- `Files/WindowsApps/Claude_1.15962.0.0_x64__pzs8sxrjxfjjc`
- `spec/worktree-dirty-cleanup.md`
- `acceptance/worktree-dirty-cleanup.md`
