# 删除提示词优化器

## 背景

管理工具曾集成第三方提示词优化器外链，并保留前端卡片、动作、Tauri 命令、旧路由兼容和产品说明。该卡片当前已不在页面中实际渲染，但相关代码和文档仍把它描述为项目能力，增加维护成本并造成能力边界误导。

## 目标

本次包含：

- 彻底删除独立提示词优化器入口、第三方外链和专属前后端代码。
- 删除 `promptOptimizer` 旧路由兼容；旧链接按未知路由回退到概览。
- 清理专属样式、测试断言和中英文产品说明。
- 增加防回归契约，确保该功能不会被残留代码重新暴露。

本次不包含：

- 删除盘古记忆的“复制给 Agent”或“新项目启动指南”。
- 删除普通确认提示、插件修复提示或其他业务提示词。
- 修改插件、供应商、Codex 注入、更新或数据存储能力。
- 删除或迁移任何用户数据。

## 用户视角描述

用户打开管理工具后，不再看到提示词优化器入口，也不会被引导到第三方提示词优化网站。工具与插件页保持原有布局和其他功能。盘古记忆仍可生成并复制当前项目接续内容和新项目启动指南。

## 功能要求

- 前端不得保留 `PromptOptimizerCard`、`goPromptOptimizer` 或专属 URL 常量。
- Tauri 不得注册或暴露 `open_prompt_optimizer_window`，也不得保留专属 Payload 和辅助函数。
- 路由标准化不再识别 `promptOptimizer`；未知路由继续沿用现有概览回退行为。
- 专属 CSS、第三方仓库链接、服务 URL 和产品说明必须删除。
- 中英文 README 的功能编号和页面说明必须保持连续、准确。
- Windows 契约测试必须断言相关入口、命令、URL、路由和样式均不存在。

## 技术约束

- 不引入新依赖。
- 不改变现有路由回退算法，只移除 `promptOptimizer` 特例。
- 不删除盘古记忆和其他模块中正常的提示词内容。
- 只做本任务所需修改，不回滚无关工作区改动。

## 交付范围

- 本规格及对应验收文档。
- Manager React/Tauri 残留功能清理。
- Windows 契约测试和相关测试夹具更新。
- README、知识图谱说明和 UI 示例同步。
- 前端检查、定向测试、默认目录 Release 构建及本机启动验证。
