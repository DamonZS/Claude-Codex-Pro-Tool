# 概览页远程公告配置

## 背景

概览页公告标题、正文、按钮和链接当前写死在 React 组件中。每次调整模型列表或宣传文案都需要改代码、重新构建并发布客户端，不适合作为运营公告。

## 目标

- 公告内容改由远程 JSON 配置驱动。
- 配置文件保存在主仓库 `assets/config/announcement.json`。
- 修改配置并推送到 `main` 后，已安装客户端刷新概览即可读取新内容，无需重新构建。
- 网络不可用、远程文件不存在或格式无效时使用内置兜底公告。

## 功能要求

- 公告配置支持：`id`、`type`、`badge`、`title`、`description`、`buttonLabel`、`url` 和 `highlights`。
- 前端不得硬编码具体公告标题、正文、按钮文案和目标链接。
- 远程配置中的官方公告必须可以覆盖内置兜底，而不是被当作重复项丢弃。
- 公告正文按纯文本渲染，不执行远程 HTML。
- 公告链接继续通过现有外部链接命令校验，仅允许 `http` 和 `https`。
- 远程请求必须带缓存破坏参数，并提供 GitHub Raw 与 jsDelivr 两个来源。

## 当前默认公告

- 标题：`CCP官方中转站`
- 正文：`拓扑API是CCP官方中转站，主打稳定接入和划算价格，支持 GPT-5.6、GPT-5.5、Claude Fable 5、Claude Opus 4.8、gpt-image-2-4k等模型与图像能力。`
- 按钮：`拓扑API`
- 链接：`https://api.toporeduce.cn`

## 非目标

- 不提供富文本 HTML、脚本或图片注入。
- 不修改公告卡片现有色调和布局。
- 不修改拓扑 API 供应商配置。

## 交付范围

- 远程公告 JSON。
- Rust 公告读取与离线兜底。
- React 动态公告渲染。
- 单元测试、前端检查和 Manager 构建。
