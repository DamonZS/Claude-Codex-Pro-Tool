# 概览页远程公告配置

## 背景

概览页公告标题、正文、按钮和链接当前写死在 React 组件中。每次调整模型列表或宣传文案都需要改代码、重新构建并发布客户端，不适合作为运营公告。

## 目标

- 公告内容改由远程 JSON 配置驱动。
- 配置文件保存在主仓库 `assets/config/announcement.json`。
- 修改配置并推送到 `main` 后，已安装客户端刷新概览即可读取新内容，无需重新构建。
- 公告默认隐藏，用户量达到运营条件后可通过远程配置明确开启。
- 新版本也可修改随安装包编译的内置配置，在远程不可用时开启公告。
- 网络不可用、远程文件不存在或格式无效时使用当前安装版本的内置配置。
- Codex 注入弹窗的“推荐内容”与管理工具公告使用同一份远程配置和内置兜底，不再维护独立广告仓库与硬编码文案。
- 管理工具与注入弹窗统一使用“合作请联系微信”作为联系入口文案，并保证深色注入界面中的 QQ 群号清晰可读。

## 功能要求

- 公告配置根节点支持 `enabled` 布尔开关；当前内置配置默认值为 `false`。
- 远程配置成功时，以远程 `enabled` 为准：`true` 显示远程公告，`false` 隐藏公告。
- 远程明确关闭是有效结果，不得触发内置公告兜底；只有所有远程来源请求或解析失败时才读取内置配置。
- 为兼容旧配置，缺少 `enabled` 或类型不是布尔值时按 `false` 处理，避免旧公告意外重新出现。
- 公告关闭时返回空公告列表，前端不得渲染空卡片。
- 公告配置支持：`id`、`type`、`badge`、`title`、`description`、`buttonLabel`、`url` 和 `highlights`。
- 前端不得硬编码具体公告标题、正文、按钮文案和目标链接。
- 远程配置中的官方公告必须可以覆盖内置兜底，而不是被当作重复项丢弃。
- 公告正文按纯文本渲染，不执行远程 HTML。
- 公告链接继续通过现有外部链接命令校验，仅允许 `http` 和 `https`。
- 远程请求必须带缓存破坏参数，并提供 GitHub Raw 与 jsDelivr 两个来源。
- 注入推荐页使用相同的 GitHub Raw 与 jsDelivr 地址；远程读取失败时使用构建时嵌入的 `assets/config/announcement.json`。
- 注入推荐卡展示公告的 `title`、`description`、`buttonLabel`、`url` 和 `highlights`，远程配置成功时不得被旧的默认推荐覆盖。
- 公告 `enabled` 开关同时控制管理工具公告卡与注入推荐内容；关闭时两处均不展示推荐卡。

## 联系入口要求

- 管理工具“关于”页和 Codex 注入弹窗的联系入口标题统一为“合作请联系微信”。
- 注入联系页保留两个 QQ 群号与一键添加入口。
- QQ 群标签和群号必须使用适配深色控制舱的高对比文字颜色，不得依赖白色卡片背景才能看清。
- 不调整现有联系页结构、二维码和外链行为。

## 当前默认公告

- 状态：默认关闭（`enabled: false`）。
- 标题：`CCP官方中转站`
- 正文：`拓扑API是CCP官方中转站，主打稳定接入和划算价格，支持 GPT-5.6、GPT-5.5、Claude Fable 5、Claude Opus 4.8、gpt-image-2-4k等最新大模型。`
- 按钮：`拓扑API`
- 链接：`https://api.toporeduce.cn`

## 非目标

- 不提供富文本 HTML、脚本或图片注入。
- 不修改公告卡片现有色调和布局。
- 不修改拓扑 API 供应商配置。

## 交付范围

- 远程公告 JSON。
- Rust 公告读取与离线兜底。
- React 动态公告渲染。
- Codex 注入推荐同源渲染与联系入口可读性修复。
- 单元测试、前端检查和 Manager 构建。

## 运营开启方式

- 远程开启：把主仓库 `assets/config/announcement.json` 的 `enabled` 改为 `true` 并推送到 `main`；客户端刷新概览后读取远程配置。
- 远程关闭：把同一字段改回 `false`；该结果优先于已安装版本的内置配置，可用于紧急下线。
- 随版本开启：发布新版本前把内置配置的 `enabled` 改为 `true` 并重新构建；该配置只在所有远程来源失败时兜底。
