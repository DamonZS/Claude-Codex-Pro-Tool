# Codex 注入盘古控制舱重设计

## 背景

当前 Codex 注入弹窗采用粗黑描边、网点纸张、偏移阴影和 `POWER PANEL` 角标，视觉语言与常见 Codex 增强工具过于接近，不能体现 Claude Codex Pro（CCP）的本地控制室定位。实际使用中，Codex 模型选择菜单还可能以 portal 形式停留在弹窗上方，覆盖弹窗内容。

## 目标

- 将注入弹窗重设计为具有 CCP 识别度的“盘古本地控制舱”。
- 首屏明确传达本地注入、模型桥接、盘古记忆、可审查和可回退能力。
- 解决模型菜单及其他临时浮层覆盖 CCP 弹窗的问题。
- 保留现有 tab、设置开关、服务模式、推荐内容、支持和联系入口。

## 非目标

- 不修改 Codex 原生界面、模型菜单业务逻辑或官方安装文件。
- 不修改 bridge API、设置字段、盘古记忆数据库或管理工具主界面。
- 不新增远程服务、依赖或不可验证的产品能力。

## 用户视角

用户点击顶部 CCP 状态入口后，先看到一个明确属于 CCP 的本地 AI 控制舱，而不是通用设置弹窗。控制舱说明当前为本地运行，并把模型桥接、盘古记忆和安全回退作为品牌能力展示。弹窗始终位于模型下拉菜单之上，用户可以直接操作或关闭。

## 功能要求

- 继续通过 `openClaudeCodexProModal()` 打开。
- 打开前主动请求 Codex 关闭当前临时菜单，并让活动元素失焦。
- 遮罩使用浏览器可用的最高 z-index，弹窗创建在文档末尾且具有独立 stacking context。
- `控制台`、`推荐内容`、`支持项目`、`联系` 四个入口继续映射现有 panel。
- 所有 `data-claude-codex-pro-setting`、后端设置、服务模式、修复、DevTools 和外部链接保持可用。
- 关闭按钮和点击遮罩只关闭弹窗，不修改设置。

## UI / 交互要求

- 采用“盘古本地控制舱”语言：深色控制台骨架、青绿主色、琥珀诊断色、低对比网格和能量轨。
- 禁止继续使用漫画字体、粗黑偏移阴影、纸张网点和 `POWER PANEL` 文案。
- Header 展示 CCP 标识、`PANGU LOCAL CONTROL DECK`、版本和后端状态。
- 左侧为场景导航，右侧为可滚动工作区；窄屏自动改为横向导航和单列内容。
- 控制台首屏展示“本机运行”说明及“模型桥接 / 盘古记忆 / 可审查回退”能力标识。
- 功能按“模型与插件通道”“会话与工作流”“本地运维与诊断”分区，降低长列表认知负担。
- 控件点击目标不小于 32px，关闭按钮不小于 40px，并提供清晰 focus-visible 状态。
- 尊重 `prefers-reduced-motion`。

## 数据与接口

- 不新增或修改后端接口。
- 不修改 `/settings/get`、`/settings/set`、`/backend/status` 等 bridge 路由。
- 品牌能力标识仅描述仓库已有功能，不伪造实时状态。

## 技术约束

- 仅修改注入脚本、定向测试和相关设计/规格文档。
- 不引入 UI 库、字体或网络资源。
- 不回滚或改动其他未提交内容。
