# 供应商模型映射下拉选择修复

## 背景

供应商编辑页从供应商接口获取模型列表后，Claude / Claude Desktop 的“模型映射”区域仍然使用普通输入框填写“实际请求模型”。用户在截图中反馈：每一行没有下拉箭头，不能直接从已获取模型中选择；同时标题说明与下方列不完全对应，容易把“显示名称”“实际请求模型”“声明支持 1M”混淆。

## 目标

本次修复包含：

- 在 Claude / Claude Desktop 供应商编辑页的模型映射中，为“实际请求模型”列提供下拉选择控件。
- 下拉选项来自已获取的供应商模型列表，并兼容当前 `modelList`、默认模型和既有映射值。
- 用户选择某个模型后，必须写入该行 `requestModel`，并同步更新 `modelMappingJson` / `modelMapping`。
- 模型映射标题与列头保持清晰对应：`显示名称`、`实际请求模型`、`声明支持 1M`。

## 非目标

- 不改供应商导入和 API Key 保存逻辑。
- 不改路由开关规则。
- 不重构整个供应商页面。
- 不改变 cc switch 导入后的配置写入格式。

## UI 要求

- “实际请求模型”列应显示为带下拉箭头的选择框。
- 下拉框应与当前暗色主题、输入框高度和边框风格一致。
- 如果当前映射值不在新获取的模型列表中，也必须保留为可选项，避免打开页面后丢值。
- 列头应与列内容对齐。

## 验证要求

- 前端类型检查通过。
- Vite 构建通过。
- `windows_subsystem` 中供应商页面回归测试通过，并锁定下拉控件存在。
