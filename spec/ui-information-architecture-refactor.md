# CCP 管理工具视觉与信息架构重构

## 背景

Claude Codex Pro Tool（CCP）已经从单一启动增强工具发展为覆盖 Codex App、Claude Desktop、Claude Code 与其他 MCP Agent 的本地 AI 运维控制台。当前界面虽然能够承载供应商、代理、模型、会话、记忆、插件、维护和更新等能力，但一级入口按历史功能增长形成，顶部操作密集、页面关系不清晰，卡片层级与装饰性视觉过多，不适合开发者长期、高频使用。

本次工作重建管理工具的信息层级、应用外壳和视觉系统。它是现有真实前端的渐进式重构，不是静态概念稿，也不改变业务层、IPC 或本地数据行为。

为保证可回滚，实施基线为提交 `838cf480126e25df62b975f7f709d754ab7a953a`，本地回滚分支为 `codex/backup-ui-refactor-baseline-20260714`。

## 产品定位

CCP 是 Windows 与 macOS 上的本地 AI 运维控制台，负责：

- 第三方 API 供应商、模型、Base URL、API Key、Header、Body 和上下文参数管理。
- OpenAI、Anthropic、Responses、Chat Completions 等协议转换、模型映射、本地代理、路由与故障转移。
- Codex 与 Claude Desktop 的本地启动增强、状态监控、插件、Skills、MCP、脚本和会话管理。
- 本地长期记忆、召回证据、项目接续与跨 Agent 记忆 MCP。
- 会话迁移、供应商同步、诊断、日志、更新、备份和维护工具。

供应商与代理能力只管理第三方 API、本地配置、协议转换和本地代理。产品不得表达为接管、托管、模拟或替代官方账号，也不得新增官方账号登录、订阅、支付、OAuth 托管或凭据托管流程。“恢复官方配置”只能表示本地配置状态切换。

## 目标

本次必须完成：

- 建立稳定的“左侧一级导航 + 顶部命令栏 + 主工作区 + 可选详情区”应用框架。
- 将一级导航控制在七个入口以内，使供应商、路由、客户端、会话、记忆和扩展之间的关系清晰。
- 建立浅色、深色、跟随系统三种主题和统一设计 Token。
- 用真实现有数据实现“客户端与增强”观察/操作页面；模型与协议继续在供应商编辑器中管理，不保留重复的独立一级页面。
- 将会话与盘古记忆、设置与关于更新分别组织为业务域内切换，不删除旧路由。
- 将供应商、会话、扩展、维护和设置等代表性页面迁移到更紧凑、可扫描的桌面工具视觉。
- 保留全部现有功能、数据结构、Tauri 命令、IPC 参数、返回结构和业务行为。
- 保持 Windows 与 macOS 原生窗口边界，不在一个平台模拟另一个平台。

本次不包含：

- 修改供应商写入、代理转发、注入、凭据、记忆、会话、插件、更新或安装的后端实现。
- 删除或伪造任何现有功能。
- 新增官方账号管理、订阅、支付、授权接管或云端凭据托管。
- 引入新的 UI 框架、图标库或远程字体。
- 一次性重写 `App.tsx` 的状态与 IPC 中枢。
- 迁移或删除用户数据、配置和记忆。

## 用户工作流

### 全局导航与筛选

- 用户通过左侧一级导航进入七个业务域：概览、供应商与路由、客户端与增强、会话与记忆、插件/Skills/MCP、维护与诊断、设置。
- 用户可折叠侧栏；折叠后保留图标、Tooltip 和清晰选中态。
- 顶部命令栏显示面包屑、当前页面、Agent 范围、当前供应商、代理健康状态和全局动作。
- 顶部最右侧固定显示三个带图标和文字的真实命令：“启动/重启 Codex”“启动/重启 Claude”“Claude 一键汉化”；搜索、当前供应商、代理健康等其他控件在其左侧排列。
- Agent 切换器是内容筛选器，不替代一级导航。客户端页面必须按“全部 / Codex / Claude”真实过滤内容。
- 用户可用 `Ctrl+K` 或 `Cmd+K` 打开命令搜索，按页面名或说明查找入口，并使用键盘选择。

### 供应商与路由

- 默认使用紧凑列表或表格展示供应商名称、目标应用、协议、模型、当前配置和连接状态。
- 选中供应商后展示连接详情与操作；原有新建、编辑、复制、删除、拖拽排序、启用、模型发现、连接测试、凭据诊断和 Claude Desktop 预览/写入行为保持不变。
- 编辑器按基础连接、目标应用、协议转换、模型映射、Header/Body、高级参数、验证与预览组织，并保留明确提交动作。
- API Key 默认遮罩，不在列表、日志或截图中显示完整值。

### 模型映射与协议

- 不设置独立的“模型与协议”导航或页面，避免与供应商编辑器重复。
- Codex 与 Claude 供应商的显示模型、实际请求模型、上下文和协议继续在供应商编辑器中管理。
- 两类供应商的“实际请求模型”使用同一套可读、可点击、可键盘操作的选择控件；菜单必须位于浮层之上，长模型 ID 不得与相邻行叠字。
- 浅色主题使用白色高不透明表面，深色主题使用深色表面；当前值、候选值、不可用提示和选中态均清晰可辨。

### 客户端与增强

- 使用现有 Overview、Claude Desktop、Claude 中文补丁、Watcher 与 Settings 状态展示 Codex、Claude Desktop 和 Claude Code 能力。
- 复用现有启动/重启、汉化、修复连接和相关维护动作。
- 明确区分“已安装”“已启用”“当前生效”“健康状态”，不得只依赖颜色。

### 会话、记忆、扩展、维护和设置

- 会话与记忆使用业务域内分段导航，分别进入原有会话和盘古记忆页面。
- 会话页不显示重复的“会话管理”介绍标题和说明卡；“历史会话修复”占据第一整行，下方 Codex 会话管理与 Claude 会话管理以 1:1 双栏展示，窄窗口再降为单列。
- 设置使用域内二级导航切换偏好设置与关于/更新。
- 扩展中心保留插件、Skills、MCP 的真实来源、目标 Agent、状态、风险、依赖和已有操作；Codex 插件仓库与 Claude 插件仓库以 1:1 双栏展示，窄窗口再降为单列。
- 工具、插件和 Skills 列表提高文字、状态徽标和 Agent 图标的对比度与清晰度，避免低对比浅色图标和过度灰化信息。
- 维护与诊断以状态清单、日志、时间线和明确修复动作优先，不制作装饰性仪表盘。
- 所有页面必须覆盖现有加载、空、错误、禁用、离线、部分成功、无权限和脏数据状态；已有状态语义不得丢失。

## 应用外壳要求

- 展开侧栏宽度为 232–248px，折叠宽度为 64px。
- 顶部命令栏高度为 52–56px。
- 可选详情面板建议宽度为 360–420px。
- 页面标准内边距为 24px，在最小窗口下可降至 16px。
- 最小可用窗口为 960×640；1280×800、1440×900、1920×1080 下布局稳定。
- 保留现有关键壳层类名 `ops-shell`、`ops-rail`、`ops-workspace`、`ops-topbar` 和 `ops-screen`，满足已有契约与兼容检查。
- 原路由值与外部 `claude-codex-pro-navigate` 导航事件保持有效。
- 新 UI 不再提供 `models` 页面；收到旧的 `models` 导航值时回落至 `supplier`，不影响其他原有路由。

## 视觉规范

视觉方向为“原生系统工具感 + 克制的空间玻璃材质 + 高密度 AI 控制台”，比例约为 70% 高不透明专业工作界面、20% 系统材质、10% Agent 品牌色和健康状态色。

- 字体：Windows 使用 Segoe UI，macOS 使用系统字体；模型 ID、路径、URL 和日志使用 JetBrains Mono 或系统等宽字体。
- 字号：页标题 20–24px，区块标题 16–18px，正文 14px，标签与说明 12–13px。
- 字距固定为 `0`，字体大小不随视口宽度缩放。
- 间距使用 4px 基准，主要值为 8、12、16、24、32px。
- 输入框和按钮圆角为 6px；面板与卡片不超过 8px；胶囊只用于状态、筛选和分段控件。
- 控件常规高度 36–40px；工具栏 52–56px；紧凑列表行 44–52px。
- 浅色：背景 `#F5F6F8`，表面 `#FFFFFF`，边框 `#E3E6EA`，主文字 `#17191C`，次文字 `#656B73`。
- 深色：背景 `#121416`，表面 `#191C20`，抬升层 `#20242A`，边框 `#2C3138`，主文字 `#F2F4F7`。
- 主色：`#0A84FF`；成功 `#10B981`；警告 `#F59E0B`；危险 `#EF4444`。
- 普通内容面板原则上无阴影。只有弹窗、命令面板、下拉菜单和 Toast 使用柔和阴影。
- AppShell、侧栏、顶部栏、命令面板和 Toast 可使用 72%–88% 不透明度与 24–32px `backdrop-filter`。
- 表格、表单、日志和长文本区域必须保持 94%–100% 不透明。
- 浅色主题内的输入、选择器、列表、编辑器和浮层不得残留硬编码深色背景；深色主题仍保持完整可用。
- 禁止紫蓝渐变、装饰光球、网格背景、霓虹赛博风、营销 Hero、卡片套卡片和持续装饰动画。

## 组件与状态

- 继续使用现有 Radix/shadcn 模式和 Lucide 图标。
- 熟悉工具动作使用图标按钮并提供 Tooltip；明确命令使用图标加文字。
- 二元设置使用 Switch，多选使用 Checkbox，互斥模式使用 Segmented Control，选项集合使用 Select 或 Menu。
- 当前选中使用浅蓝背景或细蓝描边；已启用使用 Switch 或文字标签；当前生效使用“当前配置”；健康状态使用状态点和文字；同步状态显示时间戳、结果和修改数量。
- 所有状态同时提供文字或图标，不只依赖颜色。
- 危险操作进入次级菜单或保留确认步骤。
- API Key 默认遮罩，并保留显示、复制和替换所需的既有能力。
- 图标按钮必须有可访问名称；键盘焦点环清晰。

## 主题、动效与跨平台

- 提供浅色、深色和跟随系统三种主题；选择持久化在本地 UI 偏好中，不调用后端写配置。
- 跟随系统模式监听 `prefers-color-scheme` 变化。
- 主题应同步 HTML 根节点和 Tauri 应用主体视觉，不改变业务状态。
- 应用品牌在导航、面包屑和可见产品标识中显示为 `CCP`；“盘古记忆”作为记忆功能名称可以保留。
- 动效限制在 140–220ms，只使用淡入、轻微位移、折叠和状态过渡。
- 遵循 `prefers-reduced-motion`，减少动态时关闭非必要过渡。
- 支持键盘导航、清晰焦点环和 WCAG AA 级别的常用文字对比度。
- Windows 和 macOS 分别保留自身原生窗口控制区、拖拽区和安全边距。

## 数据与接口要求

- 不修改现有 Tauri command 名称、参数、返回结构或触发时机。
- 不移动 `App.tsx` 中的 IPC 调用到新页面组件。
- 新页面只消费现有前端状态和 actions；无数据时显示真实空态。
- 不发起新增远程请求，不加载远程字体或背景资源。
- 不记录或显示完整 API Key、Token、私密会话全文和无关本地绝对路径。
- 不把主题、侧栏折叠或 Agent 筛选误写入业务设置。

## 技术约束

- 保持 React、TypeScript、Tailwind、Radix/shadcn、Lucide 和 Tauri 既有技术栈。
- 不新增 npm 依赖。
- 优先新增独立 AppShell、代表页面组件和最终 CSS Token 覆盖层，避免删除 9000 行旧样式导致功能回归。
- `App.tsx` 继续作为状态、刷新和 Tauri command 调用中枢。
- 保留 `memory`、`about` 等旧路由及外部导航兼容。
- 不修改 `assets/inject/`、Rust core、发布逻辑或用户数据位置，除非为修复本次重构直接造成的编译问题且经过明确审查。
- 不回滚或覆盖基线提交中的 Codex 注入、重启、凭据同步和其他业务修复。
