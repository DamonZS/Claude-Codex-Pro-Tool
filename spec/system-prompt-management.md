# 系统提示词管理

## 背景

CCP 需要在独立一级导航中集中管理 Codex 的系统提示词，并明确展示 `model_instructions_file` 当前状态。用户可以保存、导入、分类和启停 Markdown 提示词，不再手工修改 `~/.codex/config.toml`。

## 目标

- 新增“系统提示词”一级导航和完整管理页面。
- 支持本地提示词新增、编辑、删除、Markdown 导入、分类筛选和启停。
- 支持“保留原提示词”和“替换原提示词”两种启用模式。
- 识别 CCP 管理的提示词与外部提示词，并展示真实生效状态。
- 修改 Codex 配置前备份，采用同目录临时文件原子替换；停用时恢复启用前配置。

## 非目标

- 不修改供应商、代理、模型映射、Codex 注入或 Claude 配置。
- 不执行提示词内容，也不把提示词上传到远端。
- 不接管无法识别的外部提示词文件。

## 用户工作流

1. 用户从左侧导航进入“系统提示词”。
2. 页面展示当前生效提示词、加载方式和模板卡片。
3. 用户可新增提示词或导入 `.md` 文件，并为其设置名称、说明和分类。
4. 用户选择“保留原提示词”或“替换原提示词”，再启用某个模板。
5. CCP 写入托管指令文件和 Codex 配置；页面将该模板标记为“当前”。
6. 用户停用后，CCP 恢复启用前的 `model_instructions_file` 配置。

## 功能要求

- 提示词字段：稳定 ID、标题、文件名、说明、分类、Markdown 内容、创建和更新时间。
- 文件名仅允许安全 ASCII 文件名并以 `.md` 结尾，禁止路径分隔符和目录越界。
- 标题和内容必填；最多保存 200 个模板；单个文件最大 1 MiB。
- 删除当前生效模板前必须先停用。
- 导入仅接受 UTF-8 Markdown，标题默认取首个 Markdown 标题或文件名。
- 保留模式把启用前外部指令内容与当前模板组合到 CCP 托管文件；替换模式只写当前模板。
- 启用前保存原始 `model_instructions_file` 值；停用时精确恢复该值或删除 CCP 新增的键。
- 如果当前配置被外部程序改写，不自动覆盖外部值，并在状态中明确提示。
- 所有状态和配置写入使用临时文件、同步和原子替换；`config.toml` 修改前生成最近备份。
- GitHub 同步入口只接受用户明确提供的 HTTPS Markdown 原始地址，失败不影响本地模板。

## UI / 交互

- 页面不重复显示 AppShell 的路由标题带；顶部仅保留功能标题、同步 GitHub 模板、导入 Markdown和新增提示词操作。
- 状态区使用紧凑横向布局：左侧展示当前状态和实际启用方式标签，右侧使用两个独立按钮选择启用方式。
- 分类筛选位于模板列表上方；常用桌面宽度每行固定三张紧凑卡片，仅在内容区确实不足时降为两列或单列。
- 卡片展示标题、说明、分类、启用开关和当前状态；编辑、删除放在明确操作按钮中。
- 覆盖加载、空状态、错误状态、表单校验、操作中和外部配置状态。
- 使用 CCP 现有视觉令牌、Lucide 图标和通知机制，不引入新 UI 库。

## 接口

- Core 提供列表、保存、导入、删除、启用、停用和远程 Markdown 同步能力。
- Tauri 命令统一返回现有 `CommandResult<T>`。
- 前端仅通过 Tauri 命令访问文件系统。

## 交付范围

- Core 存储与配置事务、单元测试。
- Tauri 命令和命令注册。
- 前端类型、路由、导航、页面、对话框与响应式样式。
- 规格、验收文档和默认 Release 构建。
