# ADR 0001：盘古记忆的遗忘曲线与分层归档

- 状态：已接受
- 日期：2026-07-06
- 相关规格：`spec/memory-universal-agent-brain.md`（模块 B）
- 相关验收：`acceptance/memory-universal-agent-brain.md`

## 背景

盘古记忆当前只有一张平表 `memory_items`，记忆只增不减：

- 没有任何遗忘/衰减机制，`access_count` 只在查询排序时给微弱加分，从不按时间或低访问自动淡出。
- 唯一的"清理"是 `consolidate_items_into_lesson_manual` 的全表 `DELETE FROM memory_items` 重写，粗暴且会永久丢失原始记忆。

结果是：一年前记的、再没用过的临时结论，会一直进入会话注入摘要，稀释当前上下文。要让盘古记忆成为"越用越聪明"的通用 agent 大脑，必须让低价值记忆随时间可解释地淡出，同时绝不物理删除、保留可恢复能力。

阶段 1（模块 A）已完成语义检索（FTS5 全文 + 本地确定性向量 + 混合排序）。本 ADR 定义阶段 2（模块 B）：遗忘曲线 + 分层归档，含对应的前端界面改动。

## 决策

### 1. 两层模型：active + archived

`memory_items` 新增 `tier` 字段，取值 `active` / `archived`。归档是**可见性状态**，不是物理删除；归档记忆保留在同一张表内，可随时恢复。不做三层（不单独区分"工作/短期"层），避免过度设计——两层已能表达"当前有效 vs 已淡出"。

### 2. Ebbinghaus 指数衰减 + 访问增强

每条记忆有一个保留分（retention）：

```
retention = strength * exp(-Δt / τ)
```

- `Δt` = 当前时间 − `last_accessed_at`（复用现有字段，无需新增计时列）。
- 被检索命中（访问）时重置计时（`last_accessed_at = now`）并叠加强度（`access_count` 增强 strength）。
- `retention` 低于归档阈值时，记忆从 `active` 掉到 `archived`。

### 3. 衰减参数：约 30 天半衰期

- `τ` 取值使半衰期 ≈ 30 天（一条普通记忆若一直不被命中，约 30 天衰减到一半）。
- 归档阈值 ≈ 0.12（retention 降到约 12% 触发归档，对应约 90 天无命中）。
- 命中即重置并增强，频繁使用的记忆不会淡出。
- 参数以命名常量集中定义，便于后续调参，不散落魔法数字。

### 4. 读时懒惰计算 + fingerprint 防抖

不引入后台定时线程。衰减分数与归档判定在**读取路径**（status / query / session_summary）惰性计算并落地，复用现有 `STATUS_BACKFILL_FINGERPRINT` 式防抖，避免每次 UI 轮询都全表重算。

### 5. 豁免规则：manual + safety-rule + project-rule

以下记忆豁免衰减，永久留在 active 层（除非用户显式手动归档）：

- `source = manual`（用户手动固化的最高优先级记忆）。
- `category = safety-rule`。
- `category = project-rule`。

其余记忆参与衰减。

### 6. 归档可见性：注入只看 active，搜索可选含归档

- **会话注入路径**（`ranked_items_for_inject_cache` / `session_summary`）强制 `tier = active`——衰减的核心目的就是让淡出的记忆不再干扰注入上下文。
- **管理器搜索路径**默认只搜 active；用户可勾选"包含已归档"显式检索归档记忆（用于回捞旧结论）。

### 7. 手动归档 + 手动恢复（对称）

- 用户可主动把一条 active 记忆手动归档（即使未衰减到阈值）。
- 用户可把任意 archived 记忆恢复为 active（恢复时重置 `last_accessed_at`，给一个新鲜的衰减起点）。
- 归档/恢复都写 `memory_events` 审计。

### 8. 前端界面改动

- **归档展示**：记忆列表默认只显示 active；顶部加"显示已归档"开关。归档项置灰 + "已归档"标签 + "恢复"按钮。
- **强度可视化**：每条 active 记忆卡片显示一个细进度条表示 retention，悬停显示精确数值；豁免记忆显示为满并标注"常驻"。
- **手动操作**：active 卡片加"归档"按钮，archived 卡片加"恢复"按钮。

## 后果

### 正面

- 低价值记忆随时间可解释地淡出注入，当前上下文更聚焦。
- 归档而非删除，全程可恢复、可审计。
- 复用现有字段（`access_count` / `last_accessed_at`）和防抖先例，改动集中、无新增后台线程。
- 高价值记忆（manual / safety / project）永不淡出，安全边界稳定。

### 负面 / 风险

- 读时计算给查询路径增加一次衰减扫描；靠 fingerprint 防抖控制成本，条目极多时仍需关注。
- 半衰期参数是经验值，可能需要真机使用后调参（已集中为常量，便于调整）。
- 前端新增 tier 过滤 / 强度条 / 归档恢复交互，`windows_subsystem` 契约测试需相应更新。

## 备选方案（未采纳）

- **三层（工作/短期/长期）**：表达力更强但过度设计，两层已足够。
- **物理删除旧记忆**：不可恢复，违背"归档而非删除"的安全原则。
- **后台定时衰减线程**：与现有"读时懒惰计算"架构不一致，徒增复杂度和竞态面。
- **线性衰减 / 纯时间窗口**：不如指数衰减 + 访问增强能表达"越用越强、越冷越淡"。
