# 抖音 VibeCoding 大赏首篇投稿设计

## 目标

为 Claude Codex Pro Tool 准备第一篇面向开发者的抖音 VibeCoding 大赏投稿，突出项目的本地 AI 工具控制台定位、工程实现和 VibeCoding 开发过程。

## 受众与定位

- 受众：会使用 Codex、Claude、MCP 或其他 AI 编程工具的开发者。
- 叙事角度：先展示工具配置分散的痛点，再展示本地控制台的统一工作流，最后解释 Rust + Tauri + React 的实现边界。
- 视觉方向：采用“Before / After”对比，前段制造认知钩子，后段使用架构图和真实界面截图建立可信度。

## 投稿交付物

1. 标题：以 `VibeCoding大赏` 开头，避免绝对化和违规暗示。
2. 图文正文：不少于 100 字，包含项目动机、核心功能、技术栈、实际价值和本地安全边界。
3. 配图至少 3 张：
   - 配图 1：配置散落与本地控制台的 Before / After 对比。
   - 配图 2：Codex、Claude、MCP、插件、记忆、远程开发的架构全景图。
   - 配图 3：项目管理工具真实界面截图，重点展示供应商、插件中心或盘古记忆页面。
4. 视频脚本：时长不少于 1 分钟，包含痛点、操作演示、工程结构和结果回顾。
5. 固定互动元素：`#vibecoding大赏 #ai新星计划 @抖音科技`。

## 内容结构

### 开头 0-5 秒

展示“配置散落、插件分散、会话难找”的问题，并切换到一个本地控制台的统一界面。

### 中段

按开发者关心的顺序介绍：

1. 多供应商 / Profile 配置与切换。
2. Codex、Claude Desktop、插件、Skills 和 MCP 的管理入口。
3. 盘古记忆的 SQLite、本地采集、workspace 隔离和 MCP 共享。
4. Watcher、自恢复、日志、更新和 Release 工具链。

### 结尾

强调这是一个本地运维控制台，而不是官方产品或绕过工具；配置由用户主动管理，敏感值需要脱敏，安装操作需要预览、确认或备份。

## 风控约束

- 禁止使用“破解、绕过限制、无限额度、免费 API、内置 Key、免登录”等表达。
- 不展示 API Key、Bearer Token、Cookie、完整授权材料或用户私人路径。
- 不宣称与 Codex、Claude、OpenAI、Anthropic 或抖音官方存在合作关系。
- 不把活动曝光奖励写成产品效果或承诺。
- 不将生成的架构图伪装成真实运行截图；真实界面必须使用项目截图并脱敏。
- 以“本地管理、可审计、可备份、可恢复、用户主动配置”为主要表述。

## 验收标准

- 标题以 `VibeCoding大赏` 开头。
- 正文超过 100 个中文字符。
- 配图数量不少于 3 张，且至少 1 张为真实项目界面截图。
- 视频脚本可支持 60 秒以上演示。
- 固定话题和账号标记完整。
- 风控词检查通过，无敏感凭据和未经证实的官方背书。
- 内容能够准确对应本地项目实际存在的功能，不把规格规划误写成已上线能力。

## 非目标

- 不修改项目源码或产品功能。
- 不伪造活动获奖、官方推荐或流量结果。
- 不在文章中公开可直接复用的凭据、生产配置或第三方私有数据。
