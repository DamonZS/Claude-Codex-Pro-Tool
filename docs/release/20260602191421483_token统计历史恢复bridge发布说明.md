# Token 统计历史恢复 Bridge 发布说明

生成时间：2026-06-02 19:14

## 变更摘要

这次发布为 `Codex Token Usage` 插件补齐了宿主侧的历史恢复能力，解决“程序重启后本地无缓存时，插件无法从历史对话恢复每轮调用统计”的问题。

## 主要改动

1. 在 `BridgeDataService` 增加 `thread_usage_history` 路由能力。
2. 在 `LauncherDataService` 接入本地线程历史读取。
3. 在 `SQLiteStorageAdapter` 新增 `codex_thread_usage_history`：
   按线程 `rollout_path` 读取本地 `rollout-*.jsonl`，解析 `turn_context.turn_id` 与 `event_msg.token_count.info.last_token_usage`，同时保留 `total_token_usage.total_tokens` 作为 `contextUsed`。
4. 补充 bridge 路由测试和 rollout 历史解析测试。

## 生效效果

- 插件在没有 `sessionStorage`、没有页面内存账本时，仍可向宿主请求当前会话的历史调用记录。
- 历史恢复结果可按项目、会话和轮次正确隔离。
- 只展示当前会话最新一轮的恢复结果。

## 验证结果

已通过：

- `cargo test -p claude-codex-pro-core bridge_routes --manifest-path Cargo.toml`
- `cargo test -p claude-codex-pro-data storage_adapter --manifest-path Cargo.toml`

## 注意事项

该能力是 `Codex Token Usage 0.1.6` 的配套宿主支持。若只更新脚本市场仓、未更新当前宿主应用，则“重启后按历史恢复统计”不会完整生效。
