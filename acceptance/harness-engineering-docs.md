# 验收标准：Harness Engineering 工作法文档基线

验证对象：`spec/harness-engineering-docs.md`

## 验收项

1. 根目录代理指南存在。
   - 通过标准：`AGENTS.md` 存在于仓库根目录。
   - 验证证据：文件列表或文件内容读取结果。

2. 代理指南包含项目上下文。
   - 通过标准：`AGENTS.md` 描述 Claude Codex Pro Tool，并说明本仓库关键目录。
   - 验证证据：内容检查结果。

3. 代理指南强制先规格文档后代码、先验收标准后交付。
   - 通过标准：`AGENTS.md` 要求重要实现工作开始前必须存在相关规格文档和验收标准。
   - 验证证据：内容检查结果。

4. 代理指南说明规格文档结构。
   - 通过标准：`AGENTS.md` 列出合格规格文档的组成部分，包括标题、背景、目标、用户视角描述、功能要求、界面 / 交互要求、数据与接口要求、技术约束和交付范围。
   - 验证证据：内容检查结果。

5. 代理指南说明验收标准结构。
   - 通过标准：`AGENTS.md` 说明 `spec/*.md` 与 `acceptance/*.md` 的一一对应关系，并说明通过/失败标准、验证方式和证据要求。
   - 验证证据：内容检查结果。

6. 代理指南说明标准任务流程。
   - 通过标准：`AGENTS.md` 包含理解任务、总结上下文、实施开发、验证和交付阶段。
   - 验证证据：内容检查结果。

7. 代理指南说明子代理角色。
   - 通过标准：`AGENTS.md` 包含规格代理、验收代理、实现代理、测试代理和可选评审代理的职责。
   - 验证证据：内容检查结果。

8. 代理指南包含仓库验证命令。
   - 通过标准：`AGENTS.md` 包含依赖安装、本地开发、前端检查/构建、Rust 格式检查、Rust 测试和 Release 构建命令。
   - 验证证据：内容检查结果。

9. 匹配的规格文档和验收标准文件存在。
   - 通过标准：`spec/harness-engineering-docs.md` 与 `acceptance/harness-engineering-docs.md` 均存在。
   - 验证证据：文件列表。

10. 本次文档基线不修改应用源码。
    - 通过标准：本次新增或修改仅限 `AGENTS.md`、`spec/` 和 `acceptance/` 下的文档。
    - 验证证据：检查 `git status --short`，并区分本次文档改动与仓库中原本已有的无关改动。

## 必需验证

运行或执行：

```bash
git status --short
```

随后检查新增 Markdown 文件，确认满足以上验收项。由于本次只修改文档，不涉及运行时代码，不要求完整构建或测试。

## 不在范围内

- 构建安装器。
- 启动 Tauri 应用。
- 测试已有的源码改动。
- 修复无关的脏工作区改动。
